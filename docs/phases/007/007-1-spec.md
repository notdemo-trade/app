# Phase 7: Strategy Templates


## Overview

Open-Closed extensibility for trading behavior. Users customize LLM prompts, share via fork/export, track versions. No code execution - strategies are prompts only.

**Deploy outcome**: Users select/edit strategy templates, fork public strategies, test on symbols, set active for agent.


## Context

Phase 7 depends on Phase 6 (LLM Analysis) which provides the prompt system that strategy templates extend.

**Problem**: Users want different trading approaches (conservative vs aggressive, sector-specific, etc.) without forking code.

**MVP-e justification**: Prompt customization is simplest extensibility - no sandboxing, no code eval, no security concerns.

---


## Goals & Non-Goals

**Goals**:
- Prompt-based strategy customization
- Default opinionated templates (Conservative, Moderate, Aggressive)
- Custom variable support in prompts
- JSON validation for LLM outputs
- Strategy versioning with rollback
- Fork/share mechanism
- Star system
- Export/import JSON
- Integration with Phase 12 agent

**Non-Goals**:
- Code-based strategies (no JS/Python execution)
- Visual rule builder (post-MVP)
- Public marketplace UI (post-MVP - just export/import)
- Comments/discussions (post-MVP)
- Performance analytics (Phase 13 Trade Journal)
- Real-time collaboration

---


## Features

### 7.1 Strategy Templates Schema

Per-user strategy storage with versioning.

### 7.2 Default Templates

3 opinionated strategies seeded for all users:
1. **Conservative Mean Reversion** - RSI oversold, high-volume, 5-10% targets
2. **Aggressive Momentum** - Breakouts, high beta, trailing stops
3. **Crypto Volatility Scalping** - 24/7, tight stops, quick exits

### 7.3 Custom Variables

Users define variables in prompts: `{my_sentiment_score}`. Validated at runtime.

**TODO**: Validate `requiredVariables` matches variables used in prompts ({{variable_name}} syntax)

### 7.4 Template Editor UI

Monaco/textarea editor for prompt with test panel.

### 7.5 JSON Validation

Validate LLM output against TradeRecommendationSchema. Fallback on failure.

### 7.6 Strategy Versioning

Each edit creates version. Keep last 10. Rollback supported.

### 7.7 Fork Mechanism

Copy public strategies, track forkedFrom relationship.

### 7.8 Export/Import

JSON file format. Share via Discord/Phase 11 (Telegram)/GitHub.

### 7.9 Star System

GitHub-style stars. No unstar for MVP (or simple toggle).

### 7.10 Active Strategy Selection

One strategy active per user. Agent uses active strategy's prompt.

## Type Definitions

```ts
// packages/data-ops/src/strategies/types.ts

export type RiskTolerance = "conservative" | "moderate" | "aggressive"
export type StrategyTimeframe = "intraday" | "swing" | "position"
export type AssetClass = "stocks" | "crypto" | "options"

export interface CustomVariable {
  name: string
  type: "number" | "string" | "boolean"
  defaultValue: string | number | boolean
  description?: string
}

export interface StrategyParameters {
  riskTolerance: RiskTolerance
  timeframe: StrategyTimeframe
  preferredAssets: AssetClass[]
  customVariables: CustomVariable[]
}

export interface StrategyTemplate {
  id: string
  userId: string
  name: string
  description: string | null
  systemPrompt: string
  parameters: StrategyParameters
  requiredVariables: string[]
  isPublic: boolean
  isActive: boolean
  starCount: number
  forkCount: number
  forkedFrom: string | null
  version: integer
  createdAt: Date
  updatedAt: Date
}

export interface StrategyVersion {
  id: string
  strategyId: string
  version: number
  systemPrompt: string
  parameters: StrategyParameters
  createdAt: Date
  createdBy: string
}

export interface StrategyStar {
  userId: string
  strategyId: string
  starredAt: Date
}

export interface StrategyExport {
  name: string
  description: string | null
  systemPrompt: string
  parameters: StrategyParameters
  requiredVariables: string[]
  originalAuthor?: string
  exportedAt: string
  version: string
}
```
