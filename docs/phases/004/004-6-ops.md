## Alternatives Considered

### Event storage: per-user vs global

- **Per-user (chosen)**: simpler queries, clear ownership, easier cleanup
- **Global**: deduplication across users, but complex access control

### Ingestion trigger: cron vs queue

- **Cron (chosen)**: simpler setup, adequate for MVP
- **Queue**: better scaling, rate limit handling (future)

### X ingestion: filtered stream vs search polling

- **Search polling (chosen)**: compatible with CF Workers, reuses existing cron pattern, sufficient for trading signals (7-day window)
- **Filtered stream**: near real-time (~6-7s P99), but requires persistent HTTP connection incompatible with CF Workers. Future option via Durable Objects.

### Signal scoring: rule-based vs ML

- **Rule-based (chosen)**: transparent, predictable, no training needed
- **ML**: more accurate but complex, deferred to post-MVP

4. **Retention period**: 30 days fixed
5. **Max watchlist**: no limit (reasonable usage expected)
6. **Max symbols per ingest**: 10 (rate limit protection)
7. **Rate limit handling**: exponential backoff (1-10min), in-memory state
8. **Signal types**: both `sentiment` (per-message) and `trending` (hourly)
9. **Symbol validation**: validate against Alpaca assets API, cache 1hr

### Resolution: Cron Timezone & Active Users & Watchlist Max

**Cron timezone**: US market hours are 9:30 AM–4:00 PM ET. In UTC:
- EST (Nov–Mar): 14:30–21:00 UTC → cron: `*/5 14-21 * * 1-5`
- EDT (Mar–Nov): 13:30–20:00 UTC → cron: `*/5 13-20 * * 1-5`
Use the wider window `*/5 13-21 * * 1-5` to cover both DST periods. The extra runs outside market hours are no-ops (no new data).

**`getActiveUsers` criteria**:
```sql
SELECT DISTINCT u.id FROM auth_user u
JOIN watchlists w ON w.user_id = u.id
JOIN user_credentials c ON c.user_id = u.id AND c.provider = 'alpaca'
WHERE c.last_validated_at > NOW() - INTERVAL '30 days'
```
Active = has ≥1 watchlist symbol AND valid Alpaca credentials within 30 days.

**Watchlist max enforcement**: Add CHECK in `addToWatchlist` query:
```sql
-- Before INSERT, check count
SELECT COUNT(*) FROM watchlists WHERE user_id = $1
-- If count >= 50, return 400 "Watchlist limit reached (50 symbols)"
```
