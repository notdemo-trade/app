# Phase 4: Signal Gathering


## Overview

Ingest social signals from StockTwits for user-defined watchlists. Store raw events, deduplicate, aggregate signals.

**Deploy outcome**: Users add symbols to watchlist, StockTwits signals ingest on schedule, viewable in signal feed.


## Context

Phase 3 established portfolio view. Phase 4 adds market intelligence by ingesting signals from external sources.

**Source code (mahoraga)**:
- `src/providers/social/stocktwits.ts`: StockTwits client
- `src/jobs/cron.ts`: scheduled ingestion
- `migrations/0003_events_tables.sql`: events schema

**MVP strategy**: StockTwits only (public API, no auth). Reddit/SEC deferred to post-MVP. Twitter/X promoted to Phase 4.5 (early post-MVP) — pay-per-use pricing removes old subscription barrier, cashtag operator (`$AAPL`) maps directly to watchlist symbols.

---


## Goals & Non-Goals

**Goals**:
- Per-user watchlist (symbols to monitor)
- StockTwits signal ingestion (trending, sentiment)
- Raw event storage + deduplication
- Signal feed UI
- Scheduled cron ingestion
- 30-day cleanup job

**Non-Goals**:
- Reddit, SEC EDGAR (post-MVP)
- Twitter/X filtered stream (CF Workers can't hold persistent connections; cron polling instead)
- Real-time WebSocket streaming
- LLM classification (Phase 6)
- Technical analysis signals (Phase 5)

## Features

### 4.1 Events Schema

Two-layer storage: raw events → structured signals.

**Why two layers**:
- Raw: audit trail, reprocessing capability
- Structured: normalized for querying, aggregation

### 4.2 StockTwits Client

Port from mahoraga. Public API, no auth required.

**Methods**:
- `getTrendingSymbols()` - trending stocks with watchlist counts
- `getSymbolStream(symbol, limit)` - messages for specific symbol
- `analyzeSentiment(messages)` - aggregate bullish/bearish per symbol

### 4.3 Signal Aggregator

Process raw StockTwits messages into structured signals:
- Dedupe by source + source_id
- Extract symbols
- Calculate sentiment scores
- Store with confidence

### 4.4 Watchlist Schema

Per-user symbol list. Drives which symbols to poll.

### 4.5 Watchlist UI
Max watchlist symbols: 50, Cron timezone: UTC


Add/remove symbols, show current count.

### 4.6 Signal Feed UI

Live stream of recent signals per user's watchlist.

### 4.7 Cron Ingestion

CF Workers cron triggers global job → fans out to active users → ingests per-user watchlist signals.

### 4.8 Cleanup Job

Daily job deletes events older than 30 days.

## Type Definitions

```ts
// packages/data-ops/src/providers/social/types.ts

export type SignalSource = "stocktwits" | "reddit" | "twitter" | "sec"

export type SignalDirection = "bullish" | "bearish" | "neutral"

export type SignalType = "sentiment" | "trending" | "mention" | "insider" | "filing" | "influencer"

export interface RawEvent {
  id: string
  userId: string
  source: SignalSource
  sourceId: string
  rawContent: unknown
  ingestedAt: Date
}

export interface Signal {
  id: string
  userId: string
  rawEventId: string | null
  source: SignalSource
  symbol: string
  signalType: SignalType
  direction: SignalDirection
  strength: number // 0-1
  summary: string | null
  metadata: Record<string, unknown> | null
  createdAt: Date
}

export interface WatchlistItem {
  id: string
  userId: string
  symbol: string
  addedAt: Date
}

export interface IngestionState {
  id: string
  userId: string
  source: SignalSource
  lastPollAt: Date | null
  lastEventId: string | null
  errorCount: number
  lastError: string | null
}
```
