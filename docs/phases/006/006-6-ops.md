## Security Considerations

- LLM API keys encrypted at rest (AES-256-GCM from Phase 2)
- Keys never logged or exposed in responses
- Usage tracked per-user for billing/auditing
- Rate limiting on analysis endpoint (10 req/min)
- Content size limits on prompts (4000 chars max)

---


## Alternatives Considered

### SDK choice: Vercel AI SDK vs direct API calls

- **AI SDK (chosen)**: unified interface, 5 providers, streaming support
- **Direct APIs**: more control but more code duplication

### Cost tracking: real-time vs batch

- **Real-time (chosen)**: immediate feedback, simpler
- **Batch processing**: more accurate but delayed

### Response format: structured JSON vs free-form

- **Structured JSON (chosen)**: predictable parsing, typed responses
- **Free-form**: more flexible but harder to process

---


## Decisions

1. **SDK**: Vercel AI SDK for multi-provider support
2. **Pricing**: Static lookup table, updated manually
3. **Storage**: Store full analysis in DB, track usage separately
4. **Rate limit**: 10 analyses per minute per user
5. **Research**: Optional, costs more tokens
6. **Confidence**: 0-1 scale, displayed as percentage

## Agents SDK Integration Review (2026-02-20)

**Workflows + Retry**: Wrap `runFullAnalysis` in a Workflow with per-step retry. Step 1: gather context (technicals, signals, news). Step 2: `generateTradeRecommendation` with `retries: { limit: 3, delay: '5 seconds', backoff: 'exponential' }`. Step 3: optional `generateResearchReport` (same retry config). Step 4: persist results + usage. Each LLM call in its own step so a 429 from one provider retries only that call, not the whole pipeline.

**AI Gateway**: Route all LLM calls through `env.AI.gateway("trade-analysis")` for auto-fallback (OpenAI -> Anthropic -> Workers AI), request logging, caching, and rate-limit visibility. Eliminates need for custom retry-on-429 logic in most cases. AI Gateway provides per-provider metrics out of the box -- resolves the P1 gap "No error handling spec for LLM rate limits".

**Observability**: Not yet available as SDK primitive (`observability.emit()` not in worker types). Use AI Gateway logs for LLM call tracing instead. Monitor via Cloudflare dashboard.

**Impact**: Workflow retry + AI Gateway address the two biggest gaps (LLM 429 handling, model fallback). No schema or API changes required -- changes are internal to the analysis service layer.
