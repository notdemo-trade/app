# Phase 6: LLM Analysis


## Overview

AI-powered analysis of signals and market data using user's LLM provider keys. Multi-provider support (OpenAI, Anthropic, Google, xAI, DeepSeek) with per-user configuration, usage tracking, and cost estimation.

**Deploy outcome**: Users run AI analysis on signals/symbols, view recommendations, track LLM usage/costs.


## Context

Phase 5 established technical analysis. Phase 6 adds LLM-powered intelligence layer for signal interpretation and trade recommendations.

**Source code (mahoraga)**:
- `src/providers/llm/factory.ts`: provider factory
- `src/providers/llm/ai-sdk.ts`: Vercel AI SDK multi-provider
- `src/providers/llm/classifier.ts`: event classification, research reports
- `src/providers/types.ts`: LLMProvider interface

---


## Goals & Non-Goals

**Goals**:
- Port LLM factory with 5 provider support
- Per-user LLM credentials (encrypted, from Phase 2)
- Event classification prompt
- Research report generation
- Trade recommendation prompt (new)
- Store analysis results with token usage
- Track costs per user
- Analysis history view

**Non-Goals**:
- Streaming responses (batch only for MVP)
- Fine-tuning or custom models
- MCP agent integration (Phase 12
- Real-time analysis triggers

---


## Features

### 6.1 LLM Client Factory

Port multi-provider factory using Vercel AI SDK. Supports: OpenAI, Anthropic, Google, xAI, DeepSeek.

### 6.2 Per-User LLM Config

Load user's LLM credentials from encrypted storage (Phase 2). Create provider instance per-request.

### 6.3 Research Prompts

Port symbol research report generation. Combines market data, technicals, news into structured report.

### 6.4 Analyst Prompts

New trade decision prompt: given signals + technicals + research, recommend action (buy/sell/hold) with confidence and rationale.

### 6.5 Analysis Endpoint

`POST /api/analyze` - run analysis on symbol(s), store results, return recommendation.

### 6.6 Analysis Results Schema

Store analysis output with token counts, cost estimation, timestamps.

### 6.7 Analysis History UI

View past analyses with outcomes.

### 6.8 Usage Tracking UI

Show LLM token usage and estimated cost per user.

## Type Definitions

```ts
// packages/data-ops/src/providers/llm/types.ts

export type LLMProviderName = "openai" | "anthropic" | "google" | "xai" | "deepseek"

export interface LLMProviderConfig {
  provider: LLMProviderName
  apiKey: string
  model: string
  baseUrl?: string // OpenAI-compatible proxies
}

export interface CompletionParams {
  model?: string
  messages: Array<{
    role: "system" | "user" | "assistant"
    content: string
  }>
  temperature?: number
  max_tokens?: number
  response_format?: { type: "json_object" } | { type: "text" }
}

export interface CompletionResult {
  content: string
  usage: {
    prompt_tokens: number
    completion_tokens: number
    total_tokens: number
  }
}

export interface LLMProvider {
  complete(params: CompletionParams): Promise<CompletionResult>
  getAvailableProviders?(): LLMProviderName[]
}

export interface AnalysisRequest {
  symbol: string
  includeResearch?: boolean
  includeTechnicals?: boolean
  includeSignals?: boolean
}

export interface TradeRecommendation {
  action: "buy" | "sell" | "hold"
  confidence: number // 0-1
  rationale: string
  entry_price?: number
  target_price?: number
  stop_loss?: number
  position_size_pct?: number
  timeframe?: string
  risks: string[]
}

export interface AnalysisResult {
  id: string
  userId: string
  symbol: string
  timestamp: string
  recommendation: TradeRecommendation
  research?: string
  technicals?: Record<string, unknown>
  signals?: Array<{ type: string; direction: string; strength: number }>
  usage: {
    prompt_tokens: number
    completion_tokens: number
    total_tokens: number
    estimated_cost_usd: number
  }
  model: string
  provider: LLMProviderName
}
```

## Provider Model Pricing

```ts
// packages/data-ops/src/providers/llm/pricing.ts

export interface ModelPricing {
  input_per_1m: number  // USD per 1M input tokens
  output_per_1m: number // USD per 1M output tokens
}

export const MODEL_PRICING: Record<string, ModelPricing> = {
  // OpenAI
  "gpt-4o": { input_per_1m: 2.5, output_per_1m: 10 },
  "gpt-4o-mini": { input_per_1m: 0.15, output_per_1m: 0.6 },
  "o1": { input_per_1m: 15, output_per_1m: 60 },
  "o1-mini": { input_per_1m: 3, output_per_1m: 12 },

  // Anthropic
  "claude-sonnet-4-0": { input_per_1m: 3, output_per_1m: 15 },
  "claude-opus-4-1": { input_per_1m: 15, output_per_1m: 75 },
  "claude-3-5-haiku-latest": { input_per_1m: 0.8, output_per_1m: 4 },

  // Google
  "gemini-2.5-pro": { input_per_1m: 1.25, output_per_1m: 5 },
  "gemini-2.5-flash": { input_per_1m: 0.075, output_per_1m: 0.3 },
  "gemini-2.0-flash": { input_per_1m: 0.1, output_per_1m: 0.4 },

  // xAI
  "grok-4": { input_per_1m: 3, output_per_1m: 15 },
  "grok-3": { input_per_1m: 3, output_per_1m: 15 },

  // DeepSeek
  "deepseek-chat": { input_per_1m: 0.14, output_per_1m: 0.28 },
  "deepseek-reasoner": { input_per_1m: 0.55, output_per_1m: 2.19 },
}

export function estimateCost(
  model: string,
  promptTokens: number,
  completionTokens: number
): number {
  const pricing = MODEL_PRICING[model] ?? { input_per_1m: 1, output_per_1m: 2 }
  const inputCost = (promptTokens / 1_000_000) * pricing.input_per_1m
  const outputCost = (completionTokens / 1_000_000) * pricing.output_per_1m
  return inputCost + outputCost
}
```
