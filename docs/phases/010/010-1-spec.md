# Phase 10: Approval Flow


## Overview

Trade approval system for controlled execution. LLM recommendations generate approval tokens with TTL. Users approve/reject pending trades. Optional auto-approve mode for autonomous trading.

**Deploy outcome**: Users see pending trade approvals, approve/reject with one click, tokens expire automatically.


## Context

Phase 9 established risk management. Phase 10 adds human-in-the-loop approval before trades execute.

**Source code (mahoraga)**:
- `src/policy/approval.ts`: token generation/validation
- `src/storage/d1/queries/approvals.ts`: approval CRUD
- `src/mcp/agent.ts`: orders-preview → orders-submit flow

---


## Goals & Non-Goals

**Goals**:
- Approval token generation after policy pass
- Token validation before order execution
- Configurable TTL (default 5 minutes)
- Pending approvals list UI
- Approve/reject actions
- Auto-approve mode (bypass manual approval)
- Approval expiration and cleanup
- Approval history for audit

**Non-Goals**:
- Multi-signature approvals
- Role-based approval hierarchy
- Approval workflows (just approve/reject)
- Partial approvals (full order only)

---


## Features

### 10.1 Approval Token Schema

Tokens with TTL, linked to order preview and policy result.

### 10.2 Approval Generation

Policy passes → generate HMAC-signed approval token.

### 10.3 Approval Endpoint

`POST /api/approve/:id` - execute order with valid token.

### 10.4 Auto-Approve Mode

Skip manual approval when enabled. Agent trades autonomously.

### 10.5 Pending Approvals UI

List awaiting approval with order details.

### 10.6 Approve/Reject Buttons

One-click actions on pending approvals.

### 10.7 Approval Expiration

Auto-expire after TTL. Cron cleanup of expired tokens.
