# Phase 13: Trade Journal & Learning


## Overview

Track trade outcomes, capture context (signals, TA, notes), extract learned rules, display performance metrics.

**Deploy outcome**: Users see trade journal with P&L, can view past trades with full context, performance dashboard shows win rate and metrics.


## Context

Phase 12 autonomous agent creates trades. Phase 13 records outcomes for learning and improvement.

**Source code (mahoraga)**:
- `src/storage/d1/queries/memory.ts`: journal CRUD, rule storage, stats
- `migrations/0002_memory_tables.sql`: journal + rules schema

---


## Goals & Non-Goals

**Goals**:
- Trade journal schema (per-user)
- Auto-journal on position close
- Journal API endpoints
- Memory rules schema + extraction
- Journal list + detail UI
- Performance metrics (win rate, avg P&L)

**Non-Goals**:
- AI-powered rule extraction (manual for MVP)
- Sharpe ratio / advanced metrics (post-MVP)
- Export to CSV (post-MVP)

---


## Features

### 13.1 Trade Journal Schema

Store trade entry, exit, P&L, signals context, TA snapshot, outcome classification.

### 13.2 Auto-Journal on Close

When position closes, create journal entry linking to trade, capturing signals/TA at entry.

### 13.3 Journal Endpoint

`GET /api/journal` - list entries with filters (symbol, outcome, date range).

### 13.4 Memory Rules Schema

Store learned patterns: avoid signals, prefer setups, etc.

### 13.5 Rule Extraction

Manual rule creation from journal review (LLM-assisted in future).

### 13.6 Rule Application

Include active rules in analyst prompt context.

### 13.7 Journal List UI

Table of past trades: symbol, side, P&L, outcome, date.

### 13.8 Trade Detail UI

Full trade view: entry/exit, signals snapshot, TA at entry, notes.

### 13.9 Performance Metrics

Win rate, total P&L, avg P&L, avg hold time.

### 13.10 Performance Dashboard

Charts showing metrics over time (deferred to Phase 13).

```ts
// packages/data-ops/src/types/journal.ts

export type TradeOutcome = "win" | "loss" | "scratch"

export interface TradeJournalEntry {
  id: string
  userId: string
  tradeId: string | null
  symbol: string
  side: "buy" | "sell"
  entryPrice: number | null
  entryAt: string | null
  exitPrice: number | null
  exitAt: string | null
  qty: number
  pnlUsd: number | null
  pnlPct: number | null
  holdDurationMins: number | null
  signals: SignalSnapshot | null
  technicals: TechnicalsSnapshot | null
  regimeTags: string[]
  eventIds: string[]
  outcome: TradeOutcome | null
  notes: string | null
  lessonsLearned: string | null
  createdAt: string
  updatedAt: string
}

export interface SignalSnapshot {
  sources: Array<{
    source: string
    sentiment: number
    mentions: number
  }>
  aggregateScore: number
  topSignals: string[]
}

export interface TechnicalsSnapshot {
  rsi14: number | null
  macdHistogram: number | null
  sma20: number | null
  sma50: number | null
  atr14: number | null
  relativeVolume: number | null
}

export interface JournalStats {
  totalTrades: number
  wins: number
  losses: number
  scratches: number
  totalPnl: number
  avgPnl: number
  winRate: number
  avgHoldMins: number
}

export interface MemoryRule {
  id: string
  userId: string
  ruleType: "avoid" | "prefer" | "caution" | "insight"
  description: string
  conditions: RuleConditions | null
  confidence: number | null
  source: "manual" | "extracted"
  active: boolean
  createdAt: string
}

export interface RuleConditions {
  symbols?: string[]
  regimes?: string[]
  indicators?: Record<string, { min?: number; max?: number }>
  minWinRate?: number
  maxLoss?: number
}
```
