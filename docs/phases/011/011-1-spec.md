# Phase 11: Telegram Human-in-the-Loop Approvals


## Overview

Human-in-the-loop (HITL) trade approval system via Telegram. Users approve/reject trades from mobile or desktop. Interactive inline keyboard buttons. Timeout mechanism with auto-rejection. Broadcast notifications for trade results, daily summaries, and risk alerts.

**Deploy outcome**: Users receive trade approval requests on Telegram, tap approve/reject buttons, see trade results and daily summaries.


## Context

Phase 2 establishes credential storage including Telegram bot credentials. Phase 11 extends Phase 10 (Approval Flow) by adding Telegram bot integration for HITL approvals. Approval tokens from Phase 10 are used for Telegram workflows.

**Dependencies**:
- Phase 2: Telegram credentials (botToken, chatId)
- Phase 10: Approval flow tokens
- Phase 12: TradingAgent (Agents SDK) integration

---


## Goals & Non-Goals

**Goals**:
- Telegram bot setup wizard (create bot, get chatId)
- Pending approvals with inline keyboard (approve/reject)
- Configurable approval timeout (default 15 min)
- Auto-reject on timeout
- Notification types: approvals, results, summaries, alerts
- Notification preferences per user
- Quiet hours support
- Webhook handler for button callbacks

**Non-Goals**:
- Multiple chat destinations
- Group chat support
- Bot commands (/start handled for setup only)
- Rich media (images, charts)
- Voice/video

---


## Features

### 11.1 Telegram Bot Setup Flow

3-step wizard:
1. Create bot via @BotFather, paste botToken
2. Send /start to bot, system captures chatId
3. Test notification

### 11.2 Pending Approvals Table

Store approvals awaiting user decision.

### 11.3 Trade Approval Messages

Formatted message with trade details + inline keyboard.

### 11.4 Callback Query Handler

Process button clicks, update approval status.

### 11.5 Approval Timeout

DO alarm auto-rejects after TTL.

### 11.6 Notification Types

- `trade_approval`: Interactive approval request
- `trade_executed`: Order filled
- `trade_rejected`: User rejected or timeout
- `daily_summary`: End-of-day P&L
- `risk_alert`: Daily loss limit, kill switch
- `position_update`: Periodic position status

### 11.7 Notification Preferences

Per-user toggles for each notification type.

### 11.8 Quiet Hours

Suppress non-critical notifications during configured hours.
