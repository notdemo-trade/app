# Phase 14: Platform Stats (Tier 1 Leaderboard)


## Overview

Platform-wide aggregate statistics displayed publicly to build trust and legitimacy. No individual user data exposed.

**Deploy outcome**: Public homepage displays aggregate trading stats, users can opt-out of data inclusion.

**Reference**: sukuna.dev shows platform legitimacy through verified performance data.


## Context

Phase 13 tracks individual trade outcomes. Phase 14 aggregates anonymized stats across all opted-in users.

This is Tier 1 of a 3-tier leaderboard system:
- **Tier 1** (this): Anonymous platform aggregates (public)
- **Tier 2** (future): Individual rankings (opt-in)
- **Tier 3** (future): Verified performance badges

---


## Goals & Non-Goals

**Goals**:
- Aggregate platform statistics (users, trades, volume)
- Performance averages (ROI, win rate, Sharpe, max drawdown)
- Public display for social proof
- User opt-out mechanism
- Hourly refresh via cron
- No individual user identification

**Non-Goals**:
- Individual user rankings (Tier 2)
- Verified performance badges (Tier 3)
- Historical charts (post-MVP)
- Comparative analytics (post-MVP)
- Downloadable reports (post-MVP)

---


## Features

### 14.1 Aggregate Statistics Collection

- Total registered users
- Active agents (traded in last 7d)
- Total trades (lifetime, 30d, 7d, 24h)
- Total volume (lifetime, 30d)
- Average metrics (ROI, win rate, Sharpe, max drawdown)
- Asset class breakdown (stocks %, crypto %, options %)

### 14.2 Statistics Calculation Service

- Hourly cron job aggregates from trade_journal
- Excludes users who opted out
- Caches in platform_stats table
- Incremental updates for performance

### 14.3 User Opt-Out

- Settings toggle: "Include my data in platform statistics"
- Default: included (clear notice on signup)
- Immediate effect on next calculation cycle
- Can opt back in anytime

### 14.4 Public Stats Endpoint

- GET /api/platform/stats - no auth required
- Returns cached aggregates
- Rate limited (100 req/min per IP)

### 14.5 Stats Display Widget

- Homepage hero section
- 6 key metrics in grid
- "Last updated" timestamp
- Minimal, clean design

### 14.6 Privacy Safeguards

- Minimum 10 users required before showing stats
- Fuzzy rounding (show ~250 users, not 247)
- No breakdowns that could identify individuals
- Clear privacy policy link
