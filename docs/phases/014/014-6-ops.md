## Security & Privacy

- No PII in aggregates
- Minimum threshold (10 users) before displaying
- Fuzzy rounding prevents exact identification
- Clear opt-out mechanism
- No data retention after opt-out (excluded from next cycle)
- Rate limiting on public endpoint (100 req/min)
- Privacy policy link required on signup

## Verification

1. **Opt-out flow**: Toggle off -> verify excluded from next calculation
2. **Opt back in**: Toggle on -> verify included in next calculation
3. **Public endpoint**: No auth required, returns null if <10 users
4. **Stats update**: Hourly cron runs, stats refresh
5. **Fuzzy rounding**: 247 users shows as ~250
6. **Rate limiting**: >100 req/min returns 429

**API verification**:
```bash
# Get public stats (no auth)
curl "https://staging.notdemo.trade/api/platform/stats"

# Get privacy settings
curl -H "Authorization: Bearer <token>" \
  "https://staging.notdemo.trade/api/platform/privacy"

# Update privacy settings
curl -X PATCH -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"includeInPlatformStats": false}' \
  https://staging.notdemo.trade/api/platform/privacy
```

## Alternatives Considered

1. **Real-time calculation**: Too expensive, hourly sufficient
2. **Exact numbers**: Privacy concern, fuzzy rounding chosen
3. **Opt-out default**: Reduces data, opted-in default with clear notice
4. **Per-metric opt-out**: Too complex, single toggle preferred
5. **WebSocket updates**: Overkill for hourly data

---


## Decisions

1. **Hourly updates**: Balance freshness vs compute cost
2. **Opt-in by default**: Builds platform credibility, clear notice on signup
3. **Fuzzy rounding**: Prevents exact user identification
4. **Public endpoint**: No auth for marketing/SEO value
5. **Minimum 10 users**: Privacy threshold before displaying
6. **Single singleton row**: Simplest caching strategy

### Resolution: Sharpe Ratio

**Formula** (trade-level, simplified):
```
returns[] = trades.map(t => (t.exitPrice - t.entryPrice) / t.entryPrice)
avg = mean(returns)
std = stdDev(returns)
sharpe = std === 0 ? 0 : (avg / std) * sqrt(252)
```

**Where**: `packages/data-ops/src/queries/stats.ts` as `calculateSharpe(trades: Trade[]): number`

**Simplifications**:
- Risk-free rate = 0 (negligible for short-term trading)
- Annualized with sqrt(252) (trading days/year)
- Returns 0 when std=0 (no variance) or <2 trades
- Trade-level returns, not daily portfolio returns (simpler, sufficient for leaderboard ranking)

**Null handling**: Phases 14/15 aggregation should treat null Sharpe as 0 (new users with <2 trades get neutral score, not excluded).
