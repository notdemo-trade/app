## Alternatives Considered

### Chart library: lightweight-charts vs TradingView vs Recharts

- **lightweight-charts (chosen)**: fast, financial-focused, free, small bundle
- **TradingView**: full-featured but requires license for commercial
- **Recharts**: general purpose, not optimized for financial data

### TA calculation: client-side vs server-side

- **Server-side (chosen)**: consistent results, can cache, works with API
- **Client-side**: faster iteration but duplicates logic

### Indicator periods: fixed vs configurable

- **Fixed (chosen)**: simpler MVP, standard values (SMA 20/50/200, RSI 14, etc.)
- **Configurable**: more flexible but adds complexity (deferred)

---


## Decisions

1. **Chart library**: lightweight-charts (TradingView's open-source lib)
2. **Indicator periods**: fixed defaults (no user config for MVP)
3. **Bar limit**: 250 bars default (sufficient for all indicators)
4. **Timeframes**: 1Min, 5Min, 15Min, 1Hour, 1Day
5. **Batch limit**: 20 symbols max per request
6. **Cache**: 1 min stale time for queries
