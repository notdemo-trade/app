# Phase 16: Notifications


## Overview

Push alerts to users via Telegram, integrated with Phase 11 HITL approvals system.

**Deploy outcome**: Users receive Telegram notifications for trade executions, stop losses, errors, and daily summaries.

**Note**: This phase extends the Telegram notification system established in Phase 11. For HITL trade approvals, see `011-phase-11-telegram-approvals.md`.


## Context

Phase 13 tracks trade outcomes. Phase 16 notifies users of important events in real-time.

**Source code (mahoraga)**:
- `src/durable-objects/mahoraga-harness.ts:3341-3425`: Notification dispatch patterns (reference implementation)
- Agent now uses Agents SDK — see Phase 12 for updated TradingAgent class

---


## Goals & Non-Goals

**Goals**:
- Per-user notification config (Telegram bot)
- Telegram integration (extends Phase 11)
- Event types: trade_exec, stop_loss, error, daily_summary
- Notification log
- Preferences UI (enable/disable event types)
- Cooldown to prevent spam

**Non-Goals**:
- Discord integration (replaced by Telegram)
- Email notifications
- Push notifications (browser)
- Multiple chat destinations

---


## Features

### 16.1 Notification Config Schema

Per-user Telegram config and preferences (shares `notification_settings` table from Phase 11).

### 16.2 Telegram Integration

Send formatted HTML messages via Telegram Bot API (extends Phase 11 TelegramService).

### 16.3 Notification Events

- `trade_executed`: Order filled
- `stop_loss_triggered`: Stop loss hit
- `take_profit_triggered`: Take profit hit
- `kill_switch_activated`: Emergency halt
- `daily_loss_limit_hit`: Loss limit triggered
- `agent_error`: Agent encountered error
- `daily_summary`: End of day recap

### 16.4 Notification Log

Track sent notifications for audit.

### 16.5 Telegram Setup UI

Telegram setup handled in Phase 11. This phase adds additional notification types to the existing setup.

### 16.6 Notification Preferences

Toggle which events trigger notifications (extends Phase 11 `notification_settings` table).

## Type Definitions

```ts
// packages/data-ops/src/types/notifications.ts
// NOTE: Core notification types defined in Phase 11. Below extends those types.

export type NotificationChannel = "telegram"

export type NotificationEventType =
  | "trade_executed"
  | "stop_loss_triggered"
  | "take_profit_triggered"
  | "kill_switch_activated"
  | "daily_loss_limit_hit"
  | "agent_error"
  | "daily_summary"

// NotificationConfig defined in Phase 11 as NotificationSettings
// Uses notification_settings table with Telegram credentials from user_credentials

export interface NotificationPayload {
  eventType: NotificationEventType
  title: string
  description: string
  color: "success" | "warning" | "error" | "info"
  fields: Array<{
    name: string
    value: string
    inline?: boolean
  }>
  symbol?: string
  timestamp: string
}

export interface NotificationLog {
  id: string
  userId: string
  channel: NotificationChannel
  eventType: NotificationEventType
  payload: NotificationPayload
  status: "sent" | "failed" | "cooldown"
  errorMessage: string | null
  sentAt: string
}

// TelegramMessage interface defined in Phase 11
// Uses HTML parse_mode for formatting
```
