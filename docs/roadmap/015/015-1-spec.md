# Phase 15: Anonymous Leaderboard


## Overview

Anonymous leaderboard w/ individual rankings using composite scoring. Users opt-in, choose alias, track performance across multiple timeframes. Gamification via rankings without exposing real identities.

**Deploy outcome**: Users opt-in via settings, appear on public leaderboard with alias, view rank across timeframes, browse other traders' profiles.


## Context

Tier 2 of 3-tier leaderboard system. Builds on Phase 14 platform stats by adding individual competition.

**Reference**: sukuna.dev composite scoring: ROI (40%), Sharpe (30%), Win Rate (15%), Max Drawdown (15%)

---


## Goals & Non-Goals

**Goals**:
- Individual rankings with composite scoring
- Anonymous aliases (auto-generated or custom)
- Multiple timeframes (24h, 7d, 30d, 90d, all-time)
- Opt-in only (explicit consent)
- Anti-gaming via minimum trade requirements
- Real-time ranking updates on trade close
- Separate paper/live leaderboards

**Non-Goals**:
- Verified performance (Tier 3)
- Copy trading (regulatory)
- Strategy sharing from leaderboard
- Prize pools / rewards (post-MVP)
- Team competitions (post-MVP)
- WebSocket real-time updates (post-MVP)
- Separate leaderboards by asset class (post-MVP)

## Features

### 15.1 Leaderboard Opt-In

Settings toggle to join leaderboard. Custom alias (3-20 chars, alphanumeric + underscore). Auto-generated fallback (Agent_X7K9). Can opt-out anytime (ranking persists but hidden).

### 15.2 Composite Score Calculation

```
compositeScore =
  (ROI_normalized * 0.40) +
  (sharpe_normalized * 0.30) +
  (winRate_normalized * 0.15) +
  ((1 - maxDrawdown_normalized) * 0.15)
```

Normalization: min-max scaling across all participants in timeframe.

### 15.3 Performance Snapshots

Calculated on every trade close. Stored per timeframe. Includes: ROI, Sharpe, win rate, max drawdown, trade count. Composite score recalculated hourly (normalize against all users).

### 15.4 Minimum Requirements

- >= 5 trades in timeframe to appear
- >= 3 days trading activity
- Separate paper/live leaderboards

### 15.5 Leaderboard Display

Table: Rank, Alias, Score, ROI, Win Rate, Sharpe, Trades. Pagination (50/page). Search by alias. Highlight current user. Timeframe tabs.

### 15.6 User Profile (Anonymous)

View by alias. Shows: rank history chart, performance by timeframe. No real identity exposed.

### 15.7 Anti-Gaming Measures

- 1 leaderboard entry per Alpaca account
- Cooldown on alias changes (7 days)
- Minimum account age (7 days)

export type LeaderboardPeriod = "24h" | "7d" | "30d" | "90d" | "all"
export type TradingMode = "paper" | "live"

  userId: string
  isEnabled: boolean
  customAlias: string | null
  generatedAlias: string
  alpacaAccountId: string
  aliasLastChangedAt: string | null
  isVisible: boolean
  tradingMode: TradingMode
  createdAt: string
  updatedAt: string
}

export interface LeaderboardSnapshot {
  id: string
  userId: string
  period: LeaderboardPeriod
  tradingMode: TradingMode
  periodStart: string
  periodEnd: string
  roi: number
  sharpe: number | null
  winRate: number
  maxDrawdown: number
  tradesCount: number
  winCount: number
  lossCount: number
  totalVolume: number
  avgHoldTimeHours: number
  startingEquity: number

  compositeScore: number | null

  normalizedSharpe: number | null
  normalizedWinRate: number | null
  normalizedMaxDrawdown: number | null
  rank: number | null
  totalParticipants: number | null

}

export interface LeaderboardEntry {
  rank: number
  userId: string
  alias: string
  compositeScore: number
  roi: number
  winRate: number
  sharpe: number | null
  tradesCount: number

}

  alias: string
  ranks: Record<LeaderboardPeriod, {

    totalParticipants: number
    compositeScore: number

    tradesCount: number

    date: string

  }>
