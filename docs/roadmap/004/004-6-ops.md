## Security

- Per-user agents (StockTwits, Twitter): only access user's watchlist
- Shared agents (SEC, FRED): write public data, no user credentials
- Twitter BYOK: user's bearer token never shared
- FRED API key: platform-level, stored in env
- Rate limiting per source agent
- Raw content stored as JSONB, never executed

---

## Decisions

1. **4 agents, not 1**: each source = own DO for isolation, future billing
2. **Shared agents (SEC, FRED)**: per-ticker/series instances, data reused across users
3. **Per-user agents (StockTwits, Twitter)**: user's watchlist + credentials drive ingestion
4. **Drop Reddit**: not included in MVP
5. **SEC source**: financialdatasets.ai for filings + insider trades
6. **FRED series**: 9 macro indicators (WALCL, WTREGEN, RRPONTSYD, TGA, RRP, VIXCLS, BAMLH0A0HYM2, DTWEXBGS, DGS10)
7. **Pull model**: orchestrator calls `getSignals()`, agents don't push
8. **Shared PG**: all agents write to same tables with `source_agent` column
9. **X SDK**: `@xdevplatform/xdk`, cashtag search, no filtered stream
10. **Retention**: 30 days
