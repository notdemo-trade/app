# Phase 4: Signal Gathering Agents — Part 1: Spec

## Overview

> **Post-MVP phase** — MVP (Phase 5→6→12) runs with TA + LLM only. Adding signal agents: deploy DOs, enable entitlements, orchestrator's `gatherSignals()` schedule activates automatically.

Four standalone Cloudflare Agents, each wrapping a signal source. Each agent extends `Agent<Env, State>` with `@callable()` RPC, `scheduleEvery()` ingestion, and `this.sql` (SQLite) for local state. All agents write to shared PG `signals` + `raw_events` tables. Orchestrator (Phase 12) pulls signals via `getAgentByName()`.

**Deploy outcome**: Signal agents ingest data on schedule, write to shared PG, expose `getSignals()` RPC for orchestrator.

## Context

Phase 3 established portfolio view. Phase 4 adds market intelligence via per-source agents.

**Source code (mahoraga)**:
- `src/providers/social/stocktwits.ts`: StockTwits client
- `src/jobs/cron.ts`: scheduled ingestion
- `migrations/0003_events_tables.sql`: events schema

## Agent Definitions

| Agent | Instance | Scope | Source | API |
|-------|----------|-------|--------|-----|
| StockTwitsAgent | `{userId}` | per-user | StockTwits public API | No auth |
| TwitterAgent | `{userId}` | per-user | X API v2 (BYOK bearer) | `@xdevplatform/xdk` |
| SecFilingsAgent | `{ticker}` | shared | financialdatasets.ai | Filings + insider trades |
| FredAgent | `{seriesId}` | shared | FRED API | Macro indicators |

### Shared vs Per-User

- **Per-user** (StockTwits, Twitter): user's watchlist drives ingestion. Instance = userId.
- **Shared** (SEC, FRED): data is the same for all users. Instance = ticker/seriesId. Multiple users reference same instance. Future x402 billing per-invocation.

### Pull Model

Agents don't push signals. Orchestrator calls `agent.getSignals()` on demand. Agents write to shared PG `signals` table with `source_agent` column for cross-agent queries.

## Goals & Non-Goals

**Goals**:
- 4 standalone agents, each a Durable Object
- Per-source scheduled ingestion via `scheduleEvery()`
- Shared PG signal storage (cross-user, cross-agent)
- Agent SQLite for ingestion state, deduplication cursors
- `@callable() getSignals()` RPC for orchestrator
- SEC: filings + insider trades from financialdatasets.ai
- FRED: 9 macro series (WALCL, WTREGEN, RRPONTSYD, TGA, RRP, VIXCLS, BAMLH0A0HYM2, DTWEXBGS, DGS10)
- Watchlist UI + signal feed UI

**Non-Goals**:
- Reddit (dropped)
- Real-time WebSocket streaming
- LLM classification (Phase 6)
- Technical analysis signals (Phase 5)
- Payment/billing for shared agents (future x402)

## Features

### 4.1 StockTwitsAgent
Per-user. Public API, no auth. Polls trending + symbol streams. Extracts sentiment from user-tagged labels. `scheduleEvery(300)` (5min during market hours).

### 4.2 TwitterAgent
Per-user. BYOK bearer token. Cashtag search via `@xdevplatform/xdk`. `mention` + `influencer` signal types only (sentiment deferred to Phase 6 LLM). `scheduleEvery(300)`.

### 4.3 SecFilingsAgent
Shared, per-ticker. Polls financialdatasets.ai for SEC filings and insider trades. Signal types: `filing`, `insider`. `scheduleEvery(3600)` (hourly). Multiple users share same ticker instance.

**Endpoints**:
- `https://api.financialdatasets.ai/filings?ticker={ticker}`
- `https://api.financialdatasets.ai/insider-trades?ticker={ticker}`

### 4.4 FredAgent
Shared, per-series. Polls FRED API for macro indicator observations. Signal types: `macro`. `scheduleEvery(3600)` (hourly). One instance per series ID.

**FRED Series**:
| Series | Description |
|--------|-------------|
| WALCL | Fed balance sheet |
| WTREGEN | Treasury general account |
| RRPONTSYD | Overnight reverse repo |
| TGA | Treasury general account (alt) |
| RRP | Reverse repo |
| VIXCLS | VIX close |
| BAMLH0A0HYM2 | High yield spread |
| DTWEXBGS | Dollar index |
| DGS10 | 10Y Treasury yield |

**API**: `https://api.stlouisfed.org/fred/series/observations?series_id={id}&api_key={key}&file_type=json`

### 4.5 Watchlist UI
Add/remove symbols, max 50. Drives StockTwits/Twitter agent watchlists.

### 4.6 Signal Feed UI
Live stream of signals from all sources for user's watchlist.

### 4.7 Cleanup
30-day retention. Agents clean own SQLite. Shared PG cleanup via cron.

## Type Definitions

```ts
// packages/data-ops/src/agents/signal/types.ts

export type SignalSource = "stocktwits" | "twitter" | "sec" | "fred"

export type SignalDirection = "bullish" | "bearish" | "neutral"

export type SignalType =
  | "sentiment"
  | "trending"
  | "mention"
  | "influencer"
  | "filing"
  | "insider"
  | "macro"

export interface RawEvent {
  id: string
  sourceAgent: SignalSource
  sourceId: string
  symbol: string | null
  seriesId: string | null
  rawContent: unknown
  ingestedAt: Date
}

export interface Signal {
  id: string
  sourceAgent: SignalSource
  symbol: string | null
  seriesId: string | null
  signalType: SignalType
  direction: SignalDirection
  strength: number // 0-1
  summary: string | null
  metadata: Record<string, unknown> | null
  createdAt: Date
}

export interface WatchlistItem {
  id: string
  userId: string
  symbol: string
  addedAt: Date
}

// RPC interface all signal agents expose
export interface SignalAgentRPC {
  getSignals(since?: string): Promise<Signal[]>
  getStatus(): Promise<{ lastPollAt: string | null; errorCount: number }>
  ingestNow(): Promise<{ rawCount: number; signalCount: number }>
}

// SEC-specific
export interface SecFiling {
  ticker: string
  formType: string
  filedAt: string
  reportUrl: string
  description: string
}

export interface InsiderTrade {
  ticker: string
  ownerName: string
  transactionType: "purchase" | "sale"
  shares: number
  pricePerShare: number
  transactionDate: string
}

// FRED-specific
export interface FredObservation {
  date: string
  value: number
  seriesId: string
}
```
