## Security Considerations

- User strategies private by default
- Public strategies: read-only for others (fork to edit)
- No arbitrary code execution (prompt text only)
- LLM output validated against schema
- Rate limit strategy testing: 10 tests/minute
- Custom variables type-checked before injection
- System user owns default templates (not editable)

### XSS Prevention (P0 Blocker)

User-provided strategy content (name, description, prompts) must be sanitized before storage to prevent XSS attacks.

**Install dependency**:
```bash
pnpm add isomorphic-dompurify
```

**Sanitization utility**:
```typescript
// packages/data-ops/src/utils/sanitize.ts
import DOMPurify from 'isomorphic-dompurify'

export function sanitizeUserInput(input: string): string {
  return DOMPurify.sanitize(input, {
    ALLOWED_TAGS: [],
    ALLOWED_ATTR: []
  })
}
```

**Apply in handlers**:
```typescript
// apps/data-service/src/hono/handlers/strategies.ts
import { sanitizeUserInput } from '@repo/data-ops/utils/sanitize'

export const createStrategyHandler = async (c: Context) => {
  const body = await c.req.json()
  const validated = CreateStrategySchema.parse(body)

  const sanitized = {
    ...validated,
    name: sanitizeUserInput(validated.name),
    description: validated.description ? sanitizeUserInput(validated.description) : null,
    systemPrompt: sanitizeUserInput(validated.systemPrompt)
  }

  const strategyId = await createStrategy(c.env.DB, userId, sanitized)
  return c.json({ strategyId })
}

// Same sanitization in updateStrategyHandler
export const updateStrategyHandler = async (c: Context) => {
  const body = await c.req.json()
  const validated = UpdateStrategySchema.parse(body)

  const sanitized = {
    ...validated,
    name: validated.name ? sanitizeUserInput(validated.name) : undefined,
    description: validated.description ? sanitizeUserInput(validated.description) : undefined,
    systemPrompt: validated.systemPrompt ? sanitizeUserInput(validated.systemPrompt) : undefined
  }

  const result = await updateStrategy(c.env.DB, userId, strategyId, sanitized)
  return c.json(result)
}
```

**Sanitize before database insert**:
- Strategy name
- Strategy description
- System prompt text
- Custom variable descriptions

## Alternatives Considered

### Strategy definition: Code vs Prompts vs Config

- **Prompts (chosen)**: No security sandbox needed, accessible to non-devs, leverages existing LLM infra
- **Code (JS sandbox)**: More flexible but security risk, requires technical knowledge
- **YAML/JSON config**: More structured but less expressive, requires complex UI

### Version storage: Separate table vs JSONB array

- **Separate table (chosen)**: Cleaner queries, easier cleanup, FK integrity
- **JSONB array**: Simpler schema but harder to query/manage

### Sharing mechanism: Marketplace vs Export/Import

- **Export/Import (chosen)**: MVP-friendly, no complex UI needed, works offline
- **Marketplace**: Better UX but significant frontend work (post-MVP)

---


## Decisions

1. **Prompt-based**: Strategies are LLM prompts, not code
2. **3 default templates**: Conservative/Aggressive/Crypto - covers main use cases
3. **Custom variables**: Supported with `{name}` syntax, type-checked at runtime
4. **JSON validation**: Required for all LLM outputs, fallback on failure
5. **Versioning**: Keep last 10 versions, rollback creates new version
6. **Testing**: Use live market data via paper trading account
7. **Social**: Stars only, no comments/upvotes for MVP
8. **Export format**: JSON with version string for compatibility
