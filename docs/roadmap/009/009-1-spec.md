# Phase 9: Risk Management


## Overview

Safety layer protecting user accounts. Policy engine validates orders before execution. Kill switch for emergency halt. Daily loss limits with auto-halt. Cooldown periods after losses.

**Deploy outcome**: Users configure risk limits, orders blocked if limits exceeded, kill switch halts all trading instantly.


## Context

Phase 8 established order execution. Phase 9 adds safety checks before any order executes.

**Source code (mahoraga)**:
- `src/policy/engine.ts`: policy validation logic
- `src/policy/config.ts`: policy configuration types
- `src/policy/approval.ts`: approval token generation
- `src/storage/d1/queries/risk-state.ts`: risk state CRUD

---


## Goals & Non-Goals

**Goals**:
- Port policy engine with all validation rules
- Per-user policy configuration
- Kill switch (immediate halt)
- Daily loss tracking and limit
- Cooldown period after losses
- Pre-trade validation on every order
- Position size limits
- Max open positions limit

**Non-Goals**:
- Real-time P&L tracking (sync on order fill only)
- VaR calculations
- Portfolio-level risk analytics
- Automated position liquidation (manual via kill switch)

---


## Features

### 9.1 Policy Engine

Validate every order against configurable rules. Block if any rule violated.

### 9.2 Policy Config Schema

Per-user policy settings stored in DB.

### 9.3 Pre-Trade Validation

Check all limits before order submission.

### 9.4 Kill Switch

Emergency halt - blocks all new orders immediately.

### 9.5 Kill Switch UI

Prominent toggle, requires confirmation to disable.

### 9.6 Daily Loss Tracking

Track cumulative realized loss per day.

### 9.7 Daily Loss Limit

Auto-halt trading if daily loss exceeds threshold.

### 9.8 Cooldown Period

Pause trading after significant loss (configurable duration).

### 9.9 Take Profit Auto-Exit

Close position at target gain (optional).

### 9.10 Stop Loss Auto-Exit

Close position at max loss (optional).

### 9.11 Position Staleness

Exit positions held too long (optional, deferred).

// packages/data-ops/src/policy/types.ts

export interface PolicyConfig {
  // Position limits
  maxPositionPctEquity: number     // Max single position as % of equity (default 10%)
  maxOpenPositions: number         // Max concurrent positions (default 10)
  maxNotionalPerTrade: number      // Max $ per single trade (default 5000)

  // Daily limits
  maxDailyLossPct: number          // Max daily loss as % of equity (default 2%)
  cooldownMinutesAfterLoss: number // Pause after loss limit hit (default 30)

  // Order restrictions
  allowedOrderTypes: string[]      // market, limit, stop, stop_limit
  tradingHoursOnly: boolean        // Block outside market hours
  extendedHoursAllowed: boolean    // Allow pre/post market
  allowShortSelling: boolean       // Allow selling without position

  // Symbol filters
  allowedSymbols: string[] | null  // Null = all allowed
  denySymbols: string[]            // Explicit blocklist

  // Cash management
  useCashOnly: boolean             // Don't use margin

  // Options policy (from Phase 8)
  options: OptionsPolicyConfig
}

export interface OptionsPolicyConfig {
  optionsEnabled: boolean
  maxPctPerOptionTrade: number     // Max % of equity per options trade (default 2%)
  maxTotalOptionsExposurePct: number // Max total options exposure (default 10%)
  minDte: number                   // Min days to expiration (default 30)
  maxDte: number                   // Max days to expiration (default 60)
  minDelta: number                 // Min delta for selection (default 0.3)
  maxDelta: number                 // Max delta for selection (default 0.7)
  allowedStrategies: OptionsStrategy[]
  noAveragingDown: boolean         // Block adding to losing positions
  maxOptionPositions: number       // Max concurrent option positions (default 3)
}

export type OptionsStrategy = "long_call" | "long_put"

export interface RiskState {
  killSwitchActive: boolean
  killSwitchReason: string | null
  killSwitchAt: string | null
  dailyLossUsd: number
  dailyLossResetAt: string | null
  lastLossAt: string | null
  cooldownUntil: string | null
  updatedAt: string
}

export interface PolicyViolation {
  rule: string
  message: string
  currentValue: unknown
  limitValue: unknown
}

export interface PolicyWarning {
  rule: string
  message: string
}

export interface PolicyResult {
  allowed: boolean
  violations: PolicyViolation[]
  warnings: PolicyWarning[]
  approvalToken?: string
  approvalId?: string
  expiresAt?: string
}
