## Security Considerations

- HMAC-signed tokens prevent tampering
- User ownership check on all operations
- Token single-use (marked as used after execution)
- TTL prevents stale approvals
- Kill switch checked at approval time

---


## Alternatives Considered

### Token format: JWT vs HMAC

- **HMAC (chosen)**: simpler, no decode needed, server-side validation
- **JWT**: self-contained but unnecessary complexity

### Approval storage: DB vs KV

- **DB (chosen)**: better for queries, history, relations
- **KV**: faster but limited query capability

### TTL: fixed vs configurable

- **Configurable (chosen)**: 60s-3600s range, default 5 min
- **Fixed**: simpler but less flexible

---


## Decisions

1. **Token format**: `{approvalId}.{signature}` HMAC-SHA256
2. **Default TTL**: 5 minutes (300 seconds)
3. **Cleanup frequency**: every 15 minutes
4. **Single-use**: tokens consumed on successful execution
5. **Rejection**: optional reason stored for audit
6. **Source tracking**: manual, llm, agent for analytics

## Agents SDK Integration Review (2026-02-20)

**Workflows `step.waitForEvent()` replaces entire HMAC token system.** This is the highest-value SDK integration across all phases.

**Current design**: Generate HMAC-signed token -> store in `approvals` table -> cron cleans expired -> validate token on approve -> consume token. ~200 lines of crypto + DB + cron code.

**With Workflows**:
```ts
// TradeApprovalWorkflow
async run(event: WorkflowEvent<ApprovalParams>, step: WorkflowStep) {
  const preview = await step.do('validate-policy', () => validateOrder(event.payload))

  // Pauses workflow. Resumes when sendEvent() called. Auto-rejects on timeout.
  const decision = await step.waitForEvent<{ action: 'approve' | 'reject' }>('approval', {
    type: 'trade_approval',
    timeout: '5 minutes',
  })

  if (decision.payload.action === 'approve') {
    await step.do('execute-order', { retries: { limit: 2, delay: '1 second' } }, () =>
      submitOrder(preview)
    )
  }
}

// Approve endpoint:
const instance = await env.TRADE_APPROVAL_WORKFLOW.get(approvalId)
await instance.sendEvent({ type: 'trade_approval', payload: { action: 'approve' } })
```

**Eliminates**: HMAC signing/verification, `APPROVAL_SECRET` env var, `approvals` table status tracking, expiry cron, `cleanupExpiredApprovals`, token consumption logic. Workflow handles state, timeout, and lifecycle natively.

**Keep**: `approvals` table for audit history (insert at workflow start, update at end).

**Impact**: Removes ~200 lines. Resolves gaps: "previewHash uses simple hash" (no hash needed), "HMAC secret rotation" (no secret needed), "approval countdown" (workflow timeout is authoritative).

### Resolved Questions

- **LLM calls inside Workflow steps**: Switch to AI Gateway. Unified endpoint, auto-fallback, built-in caching/rate-limit visibility. Drop Vercel AI SDK dependency for this path.
- **Workflow instance IDs**: Generate server-side UUIDs for workflow instance IDs. Don't use user-provided values — prevents ID collisions and replay attacks.
