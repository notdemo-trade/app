## Security Considerations

- Credentials decrypted once per request, never cached long-term
- Bearer token required for all portfolio endpoints
- No credential data returned in API responses
- Alpaca errors sanitized before returning to client

---


## Alternatives Considered

### Client creation: per-request vs cached

- **Per-request (chosen)**: simpler, no cache invalidation concerns
- **Cached**: faster but needs invalidation on credential update

### Data fetching: parallel vs sequential

- **Parallel (chosen)**: fetch account + positions + orders simultaneously
- **Sequential**: simpler error handling but slower

### Portfolio history: Alpaca API vs stored locally

- **Alpaca API (chosen)**: always fresh, no storage needed
- **Local storage**: faster but needs sync mechanism

---


## Decisions

1. **Auto-refresh interval**: 30s fixed via `refetchInterval: 30_000` (not user-configurable)
2. **Tab inactive behavior**: `refetchIntervalInBackground: false` ‚Äî pauses polling when tab hidden, `refetchOnWindowFocus: true` ‚Äî refetches stale data on tab return
3. **"Last updated" timestamp**: Use TanStack Query `dataUpdatedAt` from `useQuery` return. Display as relative time ("2m ago") in dashboard header
4. **Cache portfolio**: No server-side KV cache. TanStack Query handles client-side staleness (`staleTime: 30_000`)
5. **Multiple Alpaca accounts**: No, single account per user
6. **Portfolio history chart**: SKIP for now (optional, not MVP)
7. **Positions default sort**: `market_value` descending (largest position first)
8. **Orders default filter**: `status=all`, `limit=10`, sorted by `submitted_at` desc
9. **Partial failure handling**: Dashboard loader uses `Promise.allSettled` ‚Äî renders available cards, shows error boundary per failed section
10. **Market clock timezone**: Alpaca returns ISO ET strings; display in user's local timezone via `Intl.DateTimeFormat`
11. **AlpacaCredential type**: Import from `@repo/data-ops/credentials` (Phase 2). Phase 3 does not redefine
12. **Formatter utilities**: Define `formatCurrency`, `formatTime`, `formatDateTime`, `formatRelativeTime` in `apps/user-application/src/lib/formatters.ts` using `Intl` APIs
