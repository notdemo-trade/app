## Rollback Strategy

Each phase is independently deployable. Rollback = revert the commit + redeploy.

| Phase | Rollback Impact |
|-------|----------------|
| M2 | Remove new agent stubs + bindings + Workers AI factory entry. Zero impact on existing. |
| M3 | Revert TA changes. TA fetches its own data again. |
| M4 | Remove new LLM methods. Existing `analyze()` unaffected. |
| M5 | Remove orchestrator agents + v5 migration. SessionAgent falls back to simple mode. |
| M6 | Revert UI to OrchestratorAgent connection. All backend agents remain. |
| M7 | Re-add OrchestratorAgent binding. This is the only phase that's hard to reverse — coordinate carefully. |

**Key risk**: M7 removes OrchestratorAgent. Before M7, ensure all SessionAgent functionality is verified in staging. Deploy M7 to staging first, validate for 24h, then production.

## Security Considerations

| Concern | Mitigation |
|---------|-----------|
| Unauthorized agent access | Auth middleware validates bearer token before `routeAgentRequest()` |
| Broker API key exposure | Keys stored encrypted (existing Phase 2 pattern), never in DO state |
| LLM prompt injection | Structured output parsing, JSON schema validation on LLM responses |
| Order execution without approval | `needsApproval` always returns `true` for `executeTrade` tool |
| Proposal tampering | Proposals stored server-side in SQLite, client only sends approve/reject |
| WebSocket hijacking | WebSocket upgrade validates auth token, SessionAgent per-user isolation |

## Decisions Log

| Decision | Rationale | Alternative Considered |
|----------|-----------|----------------------|
| SessionAgent extends AIChatAgent | Free message persistence, `useAgentChat` hook, built-in HITL | Custom Agent with manual message storage |
| `needsApproval` for HITL | SDK-native pattern, less custom code | Custom TradeProposal approval flow |
| Debate via LLMAnalysisAgent RPC | Reuses existing LLM provider/config/billing infrastructure | Separate DebateLLMAgent |
| Per-user broker agent | Isolates credentials and order state per user | Shared broker service |
| SQLite for discussion storage | Co-located with agent, fast reads, no network hop | Shared PG |
| `@callable()` only on SessionAgent | Clean separation: client↔SessionAgent (WebSocket), agent↔agent (DO RPC) | `@callable()` on all agents |
| Iterative migration (M2-M7) | Zero downtime, each phase testable independently | Big-bang rewrite |
| Workers AI via `@ai-sdk/cloudflare` | Zero-cost fallback, no API key needed, Vercel AI SDK stays as abstraction | Direct REST calls to CF AI API |
| Workers AI as last in fallback order | Free tier has limits; prefer user's paid provider when available | Workers AI first |
| `llmProvider`/`llmModel` in SessionConfig | User controls provider choice per session, synced to LLMAnalysisAgent | Global env var or per-request provider |
