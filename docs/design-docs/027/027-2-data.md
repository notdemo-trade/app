# Phase 27: Pipeline Confidence Threshold & Outcome Confidence Recording -- Part 2: Data

## Schema Review: No Migration Required

Both bugs are logic-only fixes. The database schema already has the correct column definitions.

### DO SQLite: `pipeline_outcomes` Table (Existing)

Defined in `pipeline-orchestrator-agent.ts` `onStart()`:

```sql
CREATE TABLE IF NOT EXISTS pipeline_outcomes (
    id                TEXT PRIMARY KEY,
    session_id        TEXT NOT NULL REFERENCES pipeline_sessions(id),
    proposal_id       TEXT NOT NULL,
    symbol            TEXT NOT NULL,
    action            TEXT NOT NULL,
    confidence        REAL NOT NULL,              -- exists, but always written as 0
    ta_signals_snapshot TEXT NOT NULL DEFAULT '[]',
    realized_pnl      REAL NOT NULL,
    realized_pnl_pct  REAL NOT NULL,
    was_correct       INTEGER NOT NULL,
    resolved_at       INTEGER NOT NULL,
    created_at        INTEGER NOT NULL
);
```

The `confidence` column is `REAL NOT NULL` and already exists. The fix changes the value written from `0` to the actual recommendation confidence. No DDL change.

### DO SQLite: `trade_proposals` Table (Existing)

Defined in `session-agent.ts` `initTables()`:

```sql
CREATE TABLE IF NOT EXISTS trade_proposals (
    id TEXT PRIMARY KEY,
    thread_id TEXT NOT NULL,
    symbol TEXT NOT NULL,
    action TEXT NOT NULL,
    confidence REAL NOT NULL,                     -- stores actual confidence at creation
    ...
);
```

The proposal's `confidence` field is already populated correctly by `buildProposal()` in the pipeline orchestrator (line 478: `confidence: rec.confidence`). This is the source of truth for Fix 2.

## Data Flow: Before and After

### Fix 1: Confidence Gate

**Before (current)**:

```
SessionAgent.runPipelineAnalysis()
  --> pipeline.runPipeline({ symbol, strategyId, strategy, ... })
      --> executeStep('llm_analysis')
          --> ctx.recommendation = { action: 'buy', confidence: 0.3, ... }
      --> executeStep('risk_validation')
          --> ctx.riskValidation = { approved: true, ... }
      --> executeStep('generate_proposal')
          --> checks: riskValidation.approved? yes
          --> checks: recommendation.action !== 'hold'? yes
          --> NO confidence check
          --> ctx.proposal = buildProposal(ctx, params)   // proposal created with 0.3 confidence
  <-- returns { session, proposal }   // proposal exists
  --> SessionAgent stores proposal     // low-confidence proposal reaches user
```

**After (fixed)**:

```
SessionAgent.runPipelineAnalysis()
  --> pipeline.runPipeline({ symbol, strategyId, strategy, ..., minConfidenceThreshold: 0.7 })
      --> executeStep('llm_analysis')
          --> ctx.recommendation = { action: 'buy', confidence: 0.3, ... }
      --> executeStep('risk_validation')
          --> ctx.riskValidation = { approved: true, ... }
      --> executeStep('generate_proposal')
          --> checks: riskValidation.approved? yes
          --> checks: recommendation.action !== 'hold'? yes
          --> checks: recommendation.confidence >= 0.7? NO (0.3 < 0.7)
          --> emits system message: "Confidence 0.30 below threshold 0.70"
          --> returns (no proposal)
  <-- returns { session, proposal: null }
  --> SessionAgent: no proposal to store
```

### Fix 2: Outcome Confidence

**Before (current)**:

```
SessionAgent.resolveOutcome(outcome)
  --> computes pnl, pnlPct
  --> distributeOutcome(outcome, pnl, pnlPct)
      --> resolvedOutcome = { symbol, realizedPnl, realizedPnlPct, action }  // no confidence
      --> pipeline.recordStepOutcome(proposalId, sessionId, resolvedOutcome)
          --> INSERT INTO pipeline_outcomes (..., confidence, ...) VALUES (..., 0, ...)
                                                                             ^-- hardcoded
```

**After (fixed)**:

```
SessionAgent.resolveOutcome(outcome)
  --> computes pnl, pnlPct
  --> looks up TradeProposal for outcome.proposalId
  --> distributeOutcome(outcome, pnl, pnlPct, proposalConfidence)
      --> resolvedOutcome = { symbol, realizedPnl, realizedPnlPct, action, confidence: 0.85 }
      --> pipeline.recordStepOutcome(proposalId, sessionId, resolvedOutcome)
          --> INSERT INTO pipeline_outcomes (..., confidence, ...) VALUES (..., 0.85, ...)
                                                                             ^-- actual value
```

## Data Integrity: Existing Rows

Existing `pipeline_outcomes` rows with `confidence = 0` are not retroactively updated. These rows were created under the buggy code and their true confidence values are not recoverable from the `pipeline_outcomes` table alone.

However, the confidence value could theoretically be recovered by:
1. Looking up the `proposal_id` in each `pipeline_outcomes` row
2. Finding the corresponding `trade_proposals` row in the SessionAgent's DO SQLite
3. Reading the `confidence` column from `trade_proposals`

This backfill is explicitly a non-goal for this phase. If needed, it can be done as a one-off script after the fix is deployed. The `pipeline_scores` table (which aggregates outcomes) does not currently use the confidence value in its computations, so the impact of stale zeros is limited to ad-hoc queries.
