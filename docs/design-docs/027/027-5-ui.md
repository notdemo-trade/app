# Phase 27: Pipeline Confidence Threshold & Outcome Confidence Recording -- Part 5: UI

## No UI Changes Required

This phase is a backend-only bug fix. No frontend routes, components, server functions, or i18n keys are modified.

---

## Confirmation of Existing UI Behavior

### Proposal Display

The frontend displays proposal confidence from `TradeProposal.confidence`, which is already populated correctly by `buildProposal()` in the pipeline orchestrator. The confidence value shown to the user was never wrong -- it was only the filtering (Fix 1) and the outcome recording (Fix 2) that had bugs.

### Session Discussion Feed

Pipeline step messages (emitted via `params.onMessage`) are already displayed in the session discussion feed. The new system message for confidence-gated rejections:

```
Confidence 0.30 below threshold 0.70. No proposal generated.
```

...will appear naturally in the discussion feed using the existing `{ type: 'system' }` sender and `'proposal'` phase formatting. No new UI component or message type is needed.

### Settings: minConfidenceThreshold

The `minConfidenceThreshold` setting is already configurable in the session config UI. The user can adjust it from 0.0 to 1.0 (with 0.7 as the default). After this fix, the setting applies equally to both debate and pipeline modes, matching user expectations.
