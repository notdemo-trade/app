## Rollback Plan

### If Migration Fails

The migration only adds columns with defaults. It cannot fail unless the database is down. If it does fail, no data is lost -- retry.

### If Code Deploy Fails

The new code reads new columns that have defaults. Rolling back to the previous code version is safe -- the old code ignores the new columns (they are not in the old schema). The new columns remain in the database but are harmless.

### If Behavioral Issues Found

Each new setting has a default that matches the current hardcoded value. If a user reports unexpected behavior:

1. Check their `user_trading_config` values via DB query
2. If values are non-default, reset to defaults
3. The old hardcoded behavior is preserved when all new fields are at their defaults
