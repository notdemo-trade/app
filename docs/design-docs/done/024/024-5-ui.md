### `PersonaCard` Component

Each persona is rendered as a card with inline editing capabilities.

```
┌─────────────────────────────────────────────────┐
│ [Bull Analyst]                    [bullish] [●]  │
│ Identifies buying opportunities...               │
│                                                   │
│ ▼ System Prompt                                  │
│ ┌─────────────────────────────────────────────┐  │
│ │ You are a bullish market analyst. Your job  │  │
│ │ is to find compelling reasons to BUY the    │  │
│ │ given asset. Focus on:                      │  │
│ │ - Positive technical momentum signals       │  │
│ │ ...                                         │  │
│ └─────────────────────────────────────────────┘  │
│                                                   │
│ [Save Changes]              [Delete] (custom only)│
└─────────────────────────────────────────────────┘
```

**Elements**:

| Element | Description | Behavior |
|---------|-------------|----------|
| Display Name | Editable for custom personas, read-only for defaults | Text input |
| Bias Badge | Color-coded chip (`bullish` = green, `bearish` = red, `neutral` = gray) | Dropdown selector |
| Active Toggle | Switch component | Disabled if toggling off would violate min-active constraint |
| Role | Brief description below name | Editable text input |
| System Prompt | Collapsible textarea | Expanded on click, auto-saves on blur or Save button |
| Delete Button | Only shown for custom (`isDefault === false`) personas | Confirmation dialog |

**Bias Badge Colors** (using existing design system):

```tsx
const BIAS_STYLES = {
  bullish: 'bg-green-500/10 text-green-500 border-green-500/20',
  bearish: 'bg-red-500/10 text-red-500 border-red-500/20',
  neutral: 'bg-gray-500/10 text-gray-400 border-gray-500/20',
} as const;
```

### `PersonaCard` Implementation

```tsx
interface PersonaCardProps {
  persona: DebatePersona;
  onUpdate: (id: string, data: UpdateDebatePersonaRequest) => void;
  onDelete: (id: string) => void;
  isUpdating: boolean;
  activeCount: number;
}

function PersonaCard({ persona, onUpdate, onDelete, isUpdating, activeCount }: PersonaCardProps) {
  const [isPromptExpanded, setIsPromptExpanded] = useState(false);

  const form = useForm({
    defaultValues: {
      displayName: persona.displayName,
      role: persona.role,
      systemPrompt: persona.systemPrompt,
      bias: persona.bias,
    },
    onSubmit: async ({ value }) => {
      onUpdate(persona.id, value);
    },
  });

  const canDisable = persona.isActive && activeCount > 2;

  return (
    <Card>
      <CardHeader className="flex flex-row items-center justify-between">
        <div className="flex items-center gap-3">
          {persona.isDefault ? (
            <CardTitle className="text-base">{persona.displayName}</CardTitle>
          ) : (
            <form.Field name="displayName">
              {(field) => (
                <Input
                  value={field.state.value}
                  onChange={(e) => field.handleChange(e.target.value)}
                  className="text-base font-semibold h-8 w-48"
                />
              )}
            </form.Field>
          )}
          <form.Field name="bias">
            {(field) => (
              <Select value={field.state.value} onValueChange={field.handleChange}>
                <SelectTrigger className={cn('h-6 w-24 text-xs', BIAS_STYLES[field.state.value])}>
                  <SelectValue />
                </SelectTrigger>
                <SelectContent>
                  <SelectItem value="bullish">Bullish</SelectItem>
                  <SelectItem value="bearish">Bearish</SelectItem>
                  <SelectItem value="neutral">Neutral</SelectItem>
                </SelectContent>
              </Select>
            )}
          </form.Field>
        </div>
        <Switch
          checked={persona.isActive}
          disabled={!canDisable && persona.isActive}
          onCheckedChange={(checked) => onUpdate(persona.id, { isActive: checked })}
        />
      </CardHeader>

      <CardContent className="space-y-4">
        <form.Field name="role">
          {(field) => (
            <div className="space-y-1">
              <Label className="text-xs text-muted-foreground">Role</Label>
              <Input
                value={field.state.value}
                onChange={(e) => field.handleChange(e.target.value)}
                className="text-sm h-8"
              />
            </div>
          )}
        </form.Field>

        <Collapsible open={isPromptExpanded} onOpenChange={setIsPromptExpanded}>
          <CollapsibleTrigger className="flex items-center gap-1 text-sm text-muted-foreground hover:text-foreground">
            <ChevronRight className={cn('h-4 w-4 transition-transform', isPromptExpanded && 'rotate-90')} />
            System Prompt
            <span className="text-xs">({persona.systemPrompt.length}/2000)</span>
          </CollapsibleTrigger>
          <CollapsibleContent className="mt-2">
            <form.Field name="systemPrompt">
              {(field) => (
                <Textarea
                  value={field.state.value}
                  onChange={(e) => field.handleChange(e.target.value)}
                  rows={8}
                  maxLength={2000}
                  className="font-mono text-sm"
                />
              )}
            </form.Field>
          </CollapsibleContent>
        </Collapsible>

        <div className="flex justify-between items-center pt-2">
          <form.Subscribe selector={(s) => s.isDirty}>
            {(isDirty) =>
              isDirty ? (
                <Button
                  type="button"
                  size="sm"
                  disabled={isUpdating}
                  onClick={() => form.handleSubmit()}
                >
                  {isUpdating ? 'Saving...' : 'Save Changes'}
                </Button>
              ) : (
                <div />
              )
            }
          </form.Subscribe>

          {!persona.isDefault && (
            <Button
              variant="ghost"
              size="sm"
              className="text-destructive hover:text-destructive"
              onClick={() => onDelete(persona.id)}
            >
              <Trash2 className="h-4 w-4 mr-1" />
              Delete
            </Button>
          )}
        </div>
      </CardContent>
    </Card>
  );
}
```

### `AddPersonaDialog` Component

A dialog for creating a new persona. Opens when "Add Persona" is clicked.

```tsx
function AddPersonaDialog({
  open,
  onOpenChange,
  onSubmit,
  isSubmitting,
}: {
  open: boolean;
  onOpenChange: (open: boolean) => void;
  onSubmit: (data: CreateDebatePersonaRequest) => void;
  isSubmitting: boolean;
}) {
  const form = useForm({
    defaultValues: {
      name: '',
      displayName: '',
      role: '',
      systemPrompt: '',
      bias: 'neutral' as PersonaBias,
    },
    onSubmit: async ({ value }) => {
      onSubmit(value);
    },
  });

  return (
    <Dialog open={open} onOpenChange={onOpenChange}>
      <DialogContent>
        <DialogHeader>
          <DialogTitle>Add Persona</DialogTitle>
          <DialogDescription>
            Create a new analysis perspective for your debate sessions.
          </DialogDescription>
        </DialogHeader>

        <form onSubmit={(e) => { e.preventDefault(); form.handleSubmit(); }} className="space-y-4">
          {/* name field */}
          <form.Field name="name"
            validators={{ onChange: z.string().min(1).max(50).regex(/^[a-z][a-z0-9_]*$/) }}>
            {(field) => (
              <div className="space-y-1">
                <Label>Identifier</Label>
                <Input
                  placeholder="e.g., momentum_trader"
                  value={field.state.value}
                  onChange={(e) => field.handleChange(e.target.value)}
                />
                <p className="text-xs text-muted-foreground">
                  Lowercase letters, numbers, and underscores only.
                </p>
              </div>
            )}
          </form.Field>

          {/* displayName, role, bias, systemPrompt fields ... */}

          <DialogFooter>
            <Button type="submit" disabled={isSubmitting}>
              {isSubmitting ? 'Creating...' : 'Create Persona'}
            </Button>
          </DialogFooter>
        </form>
      </DialogContent>
    </Dialog>
  );
}
```

### `ModeratorPromptSection` Component

Collapsible section below the persona cards for editing the moderator synthesis prompt.

```tsx
function ModeratorPromptSection({
  currentPrompt,
  onUpdate,
  isUpdating,
}: {
  currentPrompt: string | null;
  onUpdate: (prompt: string | null) => void;
  isUpdating: boolean;
}) {
  const [value, setValue] = useState(currentPrompt ?? '');
  const isCustom = currentPrompt !== null;

  return (
    <Card>
      <Collapsible>
        <CardHeader>
          <CollapsibleTrigger className="flex items-center gap-2 w-full">
            <CardTitle className="text-base">Moderator Prompt</CardTitle>
            {isCustom && (
              <span className="text-xs bg-blue-500/10 text-blue-500 px-2 py-0.5 rounded">
                Custom
              </span>
            )}
          </CollapsibleTrigger>
          <CardDescription>
            Controls how the moderator synthesizes debate results into a consensus recommendation.
          </CardDescription>
        </CardHeader>
        <CollapsibleContent>
          <CardContent className="space-y-4">
            <Textarea
              value={value}
              onChange={(e) => setValue(e.target.value)}
              placeholder={DEFAULT_MODERATOR_PROMPT_PREVIEW}
              rows={6}
              maxLength={2000}
              className="font-mono text-sm"
            />
            <div className="flex gap-2">
              <Button
                size="sm"
                disabled={isUpdating}
                onClick={() => onUpdate(value || null)}
              >
                {isUpdating ? 'Saving...' : 'Save Prompt'}
              </Button>
              {isCustom && (
                <Button
                  variant="outline"
                  size="sm"
                  onClick={() => {
                    setValue('');
                    onUpdate(null);
                  }}
                >
                  Reset to Default
                </Button>
              )}
            </div>
          </CardContent>
        </CollapsibleContent>
      </Collapsible>
    </Card>
  );
}
```

### Reset Section

```tsx
function ResetSection({ onReset, isResetting }: { onReset: () => void; isResetting: boolean }) {
  const [showConfirm, setShowConfirm] = useState(false);

  return (
    <div className="border-t pt-6">
      <div className="flex items-center justify-between">
        <div>

          <p className="text-xs text-muted-foreground mt-1">

          </p>
        </div>
        <AlertDialog open={showConfirm} onOpenChange={setShowConfirm}>
          <AlertDialogTrigger asChild>
            <Button variant="outline" size="sm" className="text-destructive">
              Reset All
            </Button>
          </AlertDialogTrigger>
          <AlertDialogContent>
            <AlertDialogHeader>
              <AlertDialogTitle>Reset debate configuration?</AlertDialogTitle>
              <AlertDialogDescription>
                This will delete all custom personas, restore default persona prompts, and reset
                the moderator prompt. Persona performance history will be preserved but may no
                longer reflect the current prompts. This action cannot be undone.
              </AlertDialogDescription>
            </AlertDialogHeader>
            <AlertDialogFooter>
              <AlertDialogCancel>Cancel</AlertDialogCancel>
              <AlertDialogAction
                onClick={() => {
                  onReset();
                  setShowConfirm(false);
                }}
                disabled={isResetting}
              >
                {isResetting ? 'Resetting...' : 'Reset to Defaults'}
              </AlertDialogAction>
            </AlertDialogFooter>
          </AlertDialogContent>

      </div>
    </div>
```

## Query Key Strategy

```ts
const DEBATE_PERSONAS_QUERY_KEY = ['debate-personas'] as const;
```

All mutations (`create`, `update`, `delete`, `reset`, `updateModeratorPrompt`) invalidate this single query key, causing a refetch of the full persona list. This avoids optimistic update complexity for an infrequently-changed settings page.

## UX Principle: Minimal Config by Default

Most users should be able to use the debate system without ever visiting this page.

1. **Defaults are seeded automatically** — 3 personas created on first access via lazy seeding, no setup required
2. **System prompts are collapsed by default** — the card shows persona name, bias badge, and active toggle. Users only see the full prompt if they click to expand. This makes the page scannable without being overwhelming.
3. **Moderator prompt section is collapsed by default** — most users never need to edit it. The section shows a "Custom" badge only when the user has overridden the default.
4. **Active toggle is the primary interaction** — the simplest customization (enable/disable a persona) is a single toggle, no form submission needed
5. **"Add Persona" is secondary** — positioned below the persona list, not prominent. Advanced users will find it; casual users won't feel pressured to create personas
6. **Reset to Defaults is always available** — gives users confidence to experiment, knowing they can revert

## UI States

### Loading State

Show a spinner while `personasQuery.isLoading` is true (same pattern as `TradingConfigPage`):

```tsx
if (personasQuery.isLoading) {
  return (
    <div className="max-w-2xl mx-auto flex items-center justify-center py-12">
      <div className="animate-spin rounded-full h-8 w-8 border-b-2 border-primary" />
    </div>
  );
}
```

### Error State

Show an alert if the query fails:

```tsx
if (personasQuery.isError) {
  return (
    <Alert variant="destructive">
      Failed to load debate personas. Please try again.
    </Alert>
  );
}
```

### Mutation Error Handling

Display error messages from the API inline. The `resultToResponse` utility returns structured errors that can be displayed:

```tsx
{updateMutation.isError && (
  <Alert variant="destructive" className="mb-4">
    {updateMutation.error.message}
  </Alert>
)}
```

### Empty State

Not applicable -- lazy seeding ensures at least 3 default personas always exist after the first load.
