# Phase 26: Fix Position Size Units Mismatch — Part 5: UI

## No UI Changes Required

The position size mismatch is a backend computation bug. The frontend displays `TradeProposal.positionSizePct` as received from the WebSocket state broadcast and does not perform any position sizing calculations.

### Current UI behavior

The trade proposal card in the session view displays position size information. After the fix:

- **Before fix**: Debate fallback proposals showed `positionSizePct: 0.05` (displayed as "0.05%", which looks obviously wrong to the user but was harmless since the UI does not compute notional)
- **After fix**: All proposals consistently show `positionSizePct: 5` (displayed as "5%")

### Trading config settings page

The settings UI at `/settings/trading` allows users to configure `positionSizePctOfCash`. This value is displayed and edited as a fraction (e.g., the user enters `0.05` or the UI shows it as "5%" with a visual transform). No change is needed -- the config storage and display are correct.

### No component changes

No React components, TanStack routes, server functions, or i18n keys are modified by this fix.
