# Phase 20: SRP Multi-Agent Architecture Refactor — Part 4: API & Server Functions

## Overview

API surface for the new agent architecture. SessionAgent is the sole WebSocket entry point for the UI. Other agents are internal — accessed only via DO RPC from SessionAgent or orchestrators.

---

## WebSocket Routing

### Entry Point

All client connections go through `SessionAgent` at:

```
/agents/SessionAgent/{userId}
```

Handled by `routeAgentRequest(request, env)` in `apps/data-service/src/index.ts`.

### No Direct Client Access to Other Agents

Unlike the current architecture where the UI connects directly to `OrchestratorAgent`, the new architecture funnels everything through `SessionAgent`. This provides:
- Single connection point
- Unified message history (AIChatAgent)
- Consistent authorization boundary
- SDK-native tool approval flow

## SessionAgent WebSocket API

### Client-Callable Methods (`@callable()`)

```ts
// Lifecycle
@callable()
async start(): Promise<void>
// Enable scheduled analysis cycles

@callable()
async stop(): Promise<void>
// Disable scheduled analysis cycles

// Configuration
@callable()
async updateConfig(partial: Partial<SessionConfigInput>): Promise<SessionConfig>
// Validates via SessionConfigSchema, merges with existing
// If llmProvider or llmModel changed, syncs to LLMAnalysisAgent

@callable()
async getConfig(): Promise<SessionConfig>
// Returns current session config (includes llmProvider, llmModel)

// Status
@callable()
async getStatus(): Promise<SessionStatus>
// Returns enabled, state, config, recent threads, pending proposals

// Manual trigger
@callable()
async triggerAnalysis(symbol: string): Promise<void>
// Runs analysis cycle immediately for given symbol

// Discussion
@callable()
async getThreads(params?: { limit?: number; status?: string }): Promise<DiscussionThread[]>
// Paginated thread history

@callable()
async getThread(threadId: string): Promise<DiscussionThread>
// Single thread with all messages

// Proposals
@callable()
async getProposals(params?: { status?: string; limit?: number }): Promise<TradeProposal[]>
// Filtered proposal list

// Persona management
@callable()
async updatePersona(persona: PersonaConfig): Promise<void>
// Update or create a persona config

@callable()
async resetPersonas(): Promise<void>
// Reset to default personas
```

### Chat Interface (`onChatMessage`)

SessionAgent extends `AIChatAgent`, so it supports the standard chat interface via `useAgentChat`:

```ts
// Client sends natural language messages
// SessionAgent responds with LLM-generated text
// Tools (analyzeSymbol, executeTrade) are available to the LLM
// needsApproval on executeTrade triggers HITL flow
```

### Real-Time State Sync

`this.setState()` pushes to all connected clients:

```ts
interface SessionAgentWebSocketState {
  enabled: boolean
  lastCycleAt: number | null
  cycleCount: number
  activeThreadId: string | null
  pendingProposalCount: number
  errorCount: number
  lastError: string | null
  // Active discussion (latest thread messages for real-time feed)
  activeThread: DiscussionThread | null
}
```

When a discussion is in progress, `broadcastThread()` updates `activeThread` in state, which the client receives instantly.

## Server Functions (user-application)

### New Server Functions

```ts
// apps/user-application/src/lib/session-connection.ts

import { useAgentChat } from 'agents/ai-chat-agent'

// Hook for connecting to SessionAgent
export function useSession(userId: string) {
  const agent = useAgentChat({
    agent: 'SessionAgent',
    name: userId,
  })

  return {
    // Chat
    messages: agent.messages,
    sendMessage: agent.sendMessage,

    // State (real-time via WebSocket)
    state: agent.state as SessionAgentWebSocketState,

    // Tool approvals
    pendingApprovals: agent.messages
      .flatMap(m => m.parts)
      .filter(p => p.type === 'tool' && p.state === 'approval-required'),
    approveToolCall: (toolCallId: string) =>
      agent.addToolApprovalResponse({ id: toolCallId, approved: true }),
    rejectToolCall: (toolCallId: string) =>
      agent.addToolApprovalResponse({ id: toolCallId, approved: false }),

    // RPC (via callable methods)
    start: () => agent.call('start'),
    stop: () => agent.call('stop'),
    updateConfig: (config: Partial<SessionConfigInput>) => agent.call('updateConfig', config),
    getConfig: () => agent.call('getConfig'),
    getStatus: () => agent.call('getStatus'),
    triggerAnalysis: (symbol: string) => agent.call('triggerAnalysis', symbol),
    getThreads: (params?: { limit?: number; status?: string }) => agent.call('getThreads', params),
    getProposals: (params?: { status?: string }) => agent.call('getProposals', params),
  }
}
```

## Agent URL Routing

All agents routed via `routeAgentRequest()` in the Worker fetch handler:

```
/agents/SessionAgent/{userId}                      → SessionAgent DO
/agents/AlpacaMarketDataAgent/{userId}:{symbol}    → AlpacaMarketDataAgent DO
/agents/AlpacaBrokerAgent/{userId}                 → AlpacaBrokerAgent DO
/agents/TechnicalAnalysisAgent/{userId}:{symbol}   → TechnicalAnalysisAgent DO
/agents/LLMAnalysisAgent/{userId}                  → LLMAnalysisAgent DO
/agents/DebateOrchestratorAgent/{userId}           → DebateOrchestratorAgent DO
/agents/PipelineOrchestratorAgent/{userId}         → PipelineOrchestratorAgent DO
```

Only `/agents/SessionAgent/{userId}` accepts WebSocket upgrades from the client. Other agent URLs are used internally by `getAgentByName()`.

## Authentication

Session agent connections require authentication. The existing auth middleware flow:

1. Client authenticates via Better Auth (existing Phase 1/2 flow)
2. Bearer token included in WebSocket upgrade request
3. `SessionAgent` validates token in `onConnect()` or via middleware in fetch handler
4. User ID extracted from token → used as instance key

```ts
// In Worker fetch handler, before routeAgentRequest:
const session = await auth.api.getSession({ headers: request.headers })
if (!session && isAgentRoute(request)) {
  return new Response('Unauthorized', { status: 401 })
}
```

## Error Handling

### Agent RPC Errors

All inter-agent RPC calls wrapped with error handling:

```ts
// Pattern used in SessionAgent and orchestrators
async function safeAgentCall<T>(fn: () => Promise<T>, context: string): Promise<Result<T>> {
  try {
    const data = await fn()
    return { ok: true, data }
  } catch (error) {
    const message = error instanceof Error ? error.message : 'Unknown error'
    return { ok: false, error: new AgentRPCError(context, message) }
  }
}
```

### Client-Facing Errors

Errors in `@callable()` methods are caught and returned as structured responses, not thrown to the WebSocket connection.

```ts
@callable()
async triggerAnalysis(symbol: string): Promise<{ ok: boolean; error?: string }> {
  const result = await safeAgentCall(
    () => this.runAnalysisCycle(symbol, this.getSessionConfig(), this.getActiveStrategy()),
    `triggerAnalysis:${symbol}`
  )
  if (!result.ok) {
    this.setState({ ...this.state, errorCount: this.state.errorCount + 1, lastError: result.error.message })
    return { ok: false, error: result.error.message }
  }
  return { ok: true }
}
```
