## Security Considerations

- Master key stored in CF secrets, never in code
- Per-user key derivation limits blast radius
- IV unique per encryption (no reuse)
- Credentials never logged or returned in API responses
- Session auth required (not bearer token) for credential management
- Validation results stored, raw errors sanitized

---


## Alternatives Considered

### Encryption: AES-256-GCM vs AES-256-CBC
- **GCM (chosen)**: authenticated encryption, detects tampering
- **CBC**: requires separate HMAC

### Key derivation: HKDF vs PBKDF2
- **HKDF (chosen)**: faster, designed for key derivation from strong keys
- **PBKDF2**: slower, designed for passwords

### Storage: DB vs KV
- **DB (chosen)**: transactional, consistent, relational to user
- **KV**: faster reads but eventually consistent, no joins

---


## Decisions

1. **Credential expiration reminder**: Yes, show warning on dashboard 3 days before credential expires (for providers with expiring keys)
2. **Rate limit validation**: Deferred — session auth sufficient for now
3. **Social/notification creds**: Deferred to signal source phase (Twitter, StockTwits, Telegram)
4. **Cache decrypted creds**: Yes, cache in Durable Object memory during agent run (decrypt once per session)
4. **Custom OpenAI base URLs**: Yes, support for proxies/self-hosted (add `baseUrl` field to LLM credential schema)
