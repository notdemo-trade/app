## Data Dependencies at Runtime

The enforcement logic needs data beyond the config fields. Here is what each guard reads:

### Pre-Analysis Guards

| Guard | Config Field | Runtime Data Needed | Source |
|-------|-------------|-------------------|--------|
| G1: Market hours | `tradingHoursOnly` | `MarketClock.isOpen` | `AlpacaBrokerAgent.getClock()` |
| G2: Ticker blacklist | `tickerBlacklist` | None (filter against watchlist) | In-memory |
| G3: Ticker allowlist | `tickerAllowlist` | None (filter against watchlist) | In-memory |
| G4: Daily loss | `maxDailyLossPct` | `PortfolioHistory.profitLossPct` | `AlpacaBrokerAgent.getPortfolioHistory()` |
| G5: Cooldown | `cooldownMinutesAfterLoss` | Last resolved loss timestamp | DO SQLite `proposal_outcomes` table |

### Pre-Execution Guards

| Guard | Config Field | Runtime Data Needed | Source |
|-------|-------------|-------------------|--------|
| E1: Max notional | `maxNotionalPerTrade` | Computed `notional` value | Local calculation in `handleTradeDecision()` |
| E2: Max position value | `maxPositionValue` | Computed `notional` value | Local calculation in `handleTradeDecision()` |
| E3: Max positions | `maxPositions` | `BrokerPosition[]` count | `AlpacaBrokerAgent.getPositions()` |
| E4: Short selling | `allowShortSelling` | Position for symbol | `AlpacaBrokerAgent.getPositions()` |

## DO SQLite Tables (Read-Only)

The cooldown guard (G5) reads from the existing `proposal_outcomes` table in the SessionAgent's DO SQLite. This table is already created by `initTables()` and populated by `createOutcomeTracking()`.

### `proposal_outcomes` Table (Existing)

```sql
CREATE TABLE IF NOT EXISTS proposal_outcomes (
  id TEXT PRIMARY KEY,
  proposal_id TEXT NOT NULL,
  thread_id TEXT NOT NULL,
  orchestration_mode TEXT NOT NULL,
  orchestrator_session_id TEXT NOT NULL,
  symbol TEXT NOT NULL,
  action TEXT NOT NULL,
  entry_price REAL NOT NULL,
  entry_qty REAL NOT NULL,
  status TEXT NOT NULL DEFAULT 'tracking',   -- 'tracking' | 'resolved'
  exit_reason TEXT,
  exit_price REAL,
  realized_pnl REAL,
  realized_pnl_pct REAL,
  hold_duration_ms INTEGER,
  resolved_at INTEGER,
  created_at INTEGER NOT NULL
);
```

**Query for cooldown check (G5):**

```sql
SELECT resolved_at FROM proposal_outcomes
WHERE status = 'resolved' AND realized_pnl < 0
ORDER BY resolved_at DESC LIMIT 1
```

This returns the timestamp of the most recent resolved losing trade. If `(Date.now() - resolved_at) < cooldownMinutesAfterLoss * 60 * 1000`, the cooldown is active.

## Broker API Data Shapes

### `MarketClock` (from `AlpacaBrokerAgent.getClock()`)

```ts
interface MarketClock {
  isOpen: boolean;
  nextOpenAt: number;   // epoch ms
  nextCloseAt: number;  // epoch ms
}
```

### `PortfolioHistory` (from `AlpacaBrokerAgent.getPortfolioHistory()`)

```ts
interface PortfolioHistory {
  timestamps: number[];     // epoch seconds
  equity: number[];
  profitLoss: number[];
  profitLossPct: number[];  // fractional (e.g., -0.015 = -1.5%)
}
```

The last element of `profitLossPct` is the most recent period (today, when called during market hours). This value is compared against `maxDailyLossPct` (also fractional, default 0.02 = 2%).

### `BrokerPosition` (from `AlpacaBrokerAgent.getPositions()`)

```ts
interface BrokerPosition {
  symbol: string;
  qty: number;
  side: string;
  avgEntryPrice: number;
  currentPrice: number;
  marketValue: number;
  unrealizedPl: number;
  unrealizedPlpc: number;
}
```

## Data Flow Diagram

```
User Trading Config (PostgreSQL)
  |
  v
resolveEffectiveConfig()  -- merges with session_config, strategy, defaults
  |
  v
EffectiveConfig (all risk fields populated)
  |
  +---> triggerAnalysis()
  |       |
  |       +-- G1: tradingHoursOnly + broker.getClock()
  |       +-- G2: tickerBlacklist (filter in-memory)
  |       +-- G3: tickerAllowlist (filter in-memory)
  |       +-- G4: maxDailyLossPct + broker.getPortfolioHistory()
  |       +-- G5: cooldownMinutesAfterLoss + DO SQLite proposal_outcomes
  |       |
  |       v
  |       [filtered symbols] -> runAnalysisForSymbol() for each
  |
  +---> handleTradeDecision()
          |
          +-- E1: maxNotionalPerTrade (vs computed notional)
          +-- E2: maxPositionValue (vs computed notional)
          +-- E3: maxPositions + broker.getPositions().length
          +-- E4: allowShortSelling + broker.getPositions() for symbol
          |
          v
          broker.placeOrder() (only if all guards pass)
```
