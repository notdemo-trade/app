### Step 1: Add `cancelAllSchedules()` Helper

**Modified file:**
- `apps/data-service/src/agents/session-agent.ts`

**Change:**
Add a private method after `rescheduleAnalysisCycle()` (around line 170):

```ts
private async cancelAllSchedules(): Promise<void> {
    const existing = this.getSchedules();
    for (const s of existing) {
        await this.cancelSchedule(s.id);
    }
}
```

## Rollback Plan

If issues are discovered after deployment:

1. **Redeploy previous version**: `pnpm run deploy:staging:data-service` (or production) with the previous commit.
2. **Impact of rollback**: The old code will re-introduce the bug, but no data is lost. Leaked schedules will accumulate again as before.
3. **No data cleanup needed**: The fix only modifies schedule lifecycle management. There are no schema changes, data writes, or state mutations to revert.

## Risk Assessment

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| `cancelAllSchedules()` throws, leaving leaked schedules | Low | Low | `enabled: false` state + safety guard in `runOutcomeTrackingCycle()` prevent execution |
| `onStart()` cancels schedules during active analysis cycle | Low | Medium | Schedule cancellation is async and does not abort in-flight method execution. The running cycle completes; only future firings are prevented. |
| Concurrent `start()` / `stop()` calls race | Low | Low | Durable Object processes requests sequentially (single-threaded). No concurrency concern. |
| New schedule types added in future without updating `cancelAllSchedules()` | N/A | N/A | `cancelAllSchedules()` cancels ALL schedules with no filter. New schedule types are automatically included. |

## Post-Fix: Existing Leaked Schedules

For users who have already accumulated leaked outcome tracking schedules:

- **Automatic cleanup**: The next `onStart()` call (triggered by deployment or DO restart) will call `cancelAllSchedules()`, removing all leaked schedules.
- **No manual intervention needed**: Every DO that receives the new code will clean itself up on restart.
- **Immediate effect**: Deploying the fix to staging/production immediately resolves the issue for all sessions. Active sessions restart via `onStart()`, and the `cancelAllSchedules()` call removes accumulated schedules.
