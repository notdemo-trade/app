## Security

- LLM API keys encrypted at rest (AES-256-GCM from Phase 2)
- Keys loaded per-request from user's credentials, never stored in agent
- Usage tracked per-user
- Rate limiting: 10 req/min on analyze endpoint
- Content size limits: 4000 chars max
- Strategy template validated via Zod

---

## Decisions

1. **Agent not service**: LLMAnalysisAgent is a DO for state mgmt, usage tracking, future x402 billing
2. **Strategy as parameter**: agent is strategy-agnostic, receives template per-request from orchestrator
3. **On-demand only**: no scheduled runs — orchestrator triggers analysis
4. **SDK**: Vercel AI SDK for multi-provider support
5. **Pricing**: static lookup table
6. **Storage**: SQLite for per-request usage, PG for aggregate
7. **Rate limit**: 10/min per user

---

## Agents SDK Integration

**Workflows + Retry**: Wrap LLM calls in Workflow with per-step retry. AI Gateway for auto-fallback between providers.

**AI Gateway**: Route all LLM calls through `env.AI.gateway("trade-analysis")` for fallback, logging, caching, rate-limit visibility.
