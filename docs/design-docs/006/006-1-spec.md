# Phase 6: LLM Analysis Agent — Part 1: Spec

## Overview

LLMAnalysisAgent: per-user Cloudflare Agent wrapping Vercel AI SDK providers. Extends `Agent<Env, State>`. Strategy-agnostic — receives `StrategyTemplate` from orchestrator per-request. On-demand only (no scheduled runs). Writes analysis results to shared PG.

**Deploy outcome**: LLM agent accepts analysis requests via RPC, generates trade recommendations using user's LLM keys, writes results to shared PG.

## Context

Phase 5 established TA agents. Phase 6 adds LLM-powered intelligence layer as a standalone agent. Previously a service — now a full agent for x402 billing, state management, usage tracking.

**Source code (mahoraga)**:
- `src/providers/llm/factory.ts`: provider factory
- `src/providers/llm/ai-sdk.ts`: Vercel AI SDK multi-provider
- `src/providers/llm/classifier.ts`: event classification, research reports

## Agent Definition

| Agent | Instance | Scope | Data Source |
|-------|----------|-------|-------------|
| LLMAnalysisAgent | `{userId}` | per-user | OpenAI / Anthropic / Google / xAI / DeepSeek (via Vercel AI SDK) |

### Strategy as Parameter

LLMAnalysisAgent is strategy-agnostic. OrchestratorAgent (Phase 12) owns strategy templates and passes the active template per-request:

```
orchestrator.runAnalysis()
  → llmAgent.analyze({ signals, technicals, strategy: userStrategy })
```

Strategy template includes: risk tolerance, position sizing bias, preferred timeframe, analysis focus areas. Templates defined in Phase 7.

### On-Demand Only

No scheduled runs. Orchestrator calls `analyze()` RPC when it has aggregated signals. Agent tracks usage/costs in SQLite + shared PG.

---

## Goals & Non-Goals

**Goals**:
- Standalone LLM agent per user
- Port multi-provider factory (5 providers via Vercel AI SDK)
- Per-user LLM credentials (encrypted, from Phase 2)
- `@callable()` RPC: `analyze()`, `classifyEvent()`, `generateReport()`
- Strategy template passed per-request (not stored in agent)
- Event classification prompt
- Research report generation
- Trade recommendation prompt
- Store analysis results + token usage in shared PG
- Track costs per user in agent SQLite
- Analysis history view + usage dashboard

**Non-Goals**:
- Streaming responses (batch only for MVP)
- Fine-tuning or custom models
- Scheduled runs (on-demand from orchestrator only)
- Strategy storage (owned by orchestrator, Phase 7/12)

## Features

### 6.1 LLMAnalysisAgent
Per-user agent. Holds LLM provider config in SQLite. Exposes `analyze()`, `classifyEvent()`, `generateReport()` RPC.

### 6.2 LLM Client Factory
Port multi-provider factory using Vercel AI SDK. Supports: OpenAI, Anthropic, Google, xAI, DeepSeek.

### 6.3 Strategy-Agnostic Analysis
`analyze({ signals, technicals, strategy })` — strategy template injected per-request from orchestrator. Agent doesn't store/own strategies.

### 6.4 Trade Recommendation
JSON-structured recommendation: action, confidence, entry/target/stop, position size, timeframe, risks.

### 6.5 Event Classification
Classify raw events into structured types (earnings, merger, insider, etc.).

### 6.6 Research Report
Generate equity research report combining signals + technicals + context.

### 6.7 Usage Tracking
Token usage + cost estimation tracked in agent SQLite (per-request) and shared PG (aggregate).

### 6.8 Analysis History UI + Usage Dashboard

## Type Definitions

```ts
// packages/data-ops/src/agents/llm/types.ts

export type LLMProviderName = "openai" | "anthropic" | "google" | "xai" | "deepseek"

export interface LLMProviderConfig {
  provider: LLMProviderName
  apiKey: string
  model: string
  baseUrl?: string
}

export interface CompletionParams {
  messages: Array<{ role: "system" | "user" | "assistant"; content: string }>
  temperature?: number
  max_tokens?: number
  response_format?: { type: "json_object" } | { type: "text" }
}

export interface CompletionResult {
  content: string
  usage: { prompt_tokens: number; completion_tokens: number; total_tokens: number }
}

export interface StrategyTemplate {
  id: string
  name: string
  riskTolerance: "conservative" | "moderate" | "aggressive"
  positionSizeBias: number // 0.01-0.25
  preferredTimeframe: "intraday" | "swing" | "position"
  analysisFocus: string[] // e.g. ["momentum", "value", "macro"]
  customPromptSuffix?: string
}

export interface AnalysisRequest {
  symbol: string
  signals: Array<{ type: string; direction: string; strength: number; source: string }>
  technicals?: Record<string, unknown>
  strategy: StrategyTemplate
  includeResearch?: boolean
}

export interface TradeRecommendation {
  action: "buy" | "sell" | "hold"
  confidence: number // 0-1
  rationale: string
  entry_price?: number
  target_price?: number
  stop_loss?: number
  position_size_pct?: number
  timeframe?: string
  risks: string[]
}

export interface AnalysisResult {
  id: string
  userId: string
  symbol: string
  timestamp: string
  recommendation: TradeRecommendation
  research?: string
  strategyId: string
  usage: { prompt_tokens: number; completion_tokens: number; total_tokens: number; estimated_cost_usd: number }
  model: string
  provider: LLMProviderName
}

// Agent state
export interface LLMAgentState {
  totalAnalyses: number
  totalTokens: number
  totalCostUsd: number
  lastAnalysisAt: string | null
  errorCount: number
  lastError: string | null
}

// RPC interface
export interface LLMAgentRPC {
  analyze(request: AnalysisRequest): Promise<AnalysisResult>
  classifyEvent(rawContent: string): Promise<{ event_type: string; symbols: string[]; summary: string; confidence: number }>
  generateReport(symbol: string, context: Record<string, unknown>): Promise<{ report: string }>
  getUsage(days?: number): Promise<{ totalTokens: number; totalCostUsd: number }>
}
```
