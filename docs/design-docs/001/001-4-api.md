## API Endpoints (data-service)

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET | `/api/status` | Bearer token | Health check with user context |
| GET | `/api/tokens` | Session | List user's tokens |
| POST | `/api/tokens` | Session | Create new token |
| DELETE | `/api/tokens/:id` | Session | Revoke token |

**Note**: Token management uses session auth (from user-application), not bearer tokens.

## Rate Limiting (P0 Blocker)

Per-user rate limits prevent API abuse.

### Install Dependency

```bash
pnpm add hono-rate-limiter
```

### Middleware Implementation

```ts
// apps/data-service/src/hono/middleware/rate-limit.ts

import { rateLimiter } from 'hono-rate-limiter'

export const tokenRateLimit = rateLimiter({
  windowMs: 60_000, // 1 minute
  limit: 5,
  keyGenerator: (c) => c.get('userId') || c.req.header('cf-connecting-ip') || 'anonymous'
})
```

### Apply to Token Endpoints

```ts
// apps/data-service/src/hono/app.ts

import { tokenRateLimit } from './middleware/rate-limit'

// Apply to token creation
app.post('/api/tokens', tokenRateLimit, createTokenHandler)
```

**Limits**:
- Token generation: 5 req/min per user

**Why**: Prevent token generation abuse, protect session management endpoints.
