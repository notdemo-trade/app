**File:** `apps/data-service/src/agents/session-agent-helpers.ts`

### Manual smoke test procedure

1. Set `positionSizePctOfCash = 0.05` in trading config
2. Start a session in **debate mode** with a single watchlist symbol
3. Wait for analysis cycle to complete
4. If a proposal is generated, check `positionSizePct` in the proposal (via WebSocket state or DO SQLite query)
5. Approve the proposal
6. Verify the order notional is approximately `cash * 0.05` (e.g., ~$5,000 on $100k)
7. Stop the session
8. Start a new session in **pipeline mode** with the same config
9. Repeat steps 3-6

## Rollback Plan

If issues are discovered after deployment:

1. **Redeploy previous version**: `git revert` the fix commit and redeploy `data-service`. This restores the original (buggy) behavior, which is the known state.
2. **No data cleanup needed**: The fix does not modify any stored data. Proposals created with the fix have correct `positionSizePct` values; proposals created before the fix have the old (potentially incorrect) values. Neither requires migration.

The rollback is safe because:
- No database schema was changed
- No new tables, columns, or migrations were introduced
- The old code was functional (just had incorrect position sizes in specific fallback scenarios)
- Active sessions will pick up the redeployed code on next analysis cycle (DOs are restarted on deploy)
