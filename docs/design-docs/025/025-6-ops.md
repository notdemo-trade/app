## Rollback Plan

If issues are discovered after deployment:

1. **Redeploy previous version of `data-service`**: The new `orchestrator_session_id` column remains in existing DO SQLite tables (cannot be dropped), but the old code does not reference it. The old `resolveOrchestratorSessionId()` continues to produce wrong IDs as before -- exact same behavior as pre-fix.
2. **No data corruption risk**: The new column is nullable. The old `storeProposal()` does not include it in the INSERT, so SQLite defaults it to NULL. The old `rowToProposal()` does not read it.
3. **No user-visible change**: Persona/pipeline scores were already empty before the fix. Returning to empty scores is not a regression.

| Risk | Likelihood | Impact | Mitigation |

| Pre-fix proposals crash on null session ID | Low | Med | Explicit null check in `createOutcomeTracking()` handles gracefully |
