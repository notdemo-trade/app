# Phase 5: Technical Analysis Agent — Part 1: Spec

## Overview

TechnicalAnalysisAgent: per-user per-symbol Cloudflare Agent computing SMA/EMA/RSI/MACD/BB/ATR from Alpaca market data. Extends `Agent<Env, State>` with `@callable()` RPC and `scheduleEvery()`. Writes TA signals to shared PG. Orchestrator calls `getSignals()` via `getAgentByName()`.

**Deploy outcome**: TA agents compute indicators on schedule, write signals to shared PG, expose `getSignals()` + `getIndicators()` RPC. UI shows charts + indicators.

## Context

Phase 4 established signal gathering agents. Phase 5 adds market-based technical signals as a standalone agent.

**Source code (mahoraga)**:
- `src/providers/technicals.ts`: indicator calculations, signal detection
- `src/providers/alpaca/market-data.ts`: bars/quotes fetching

---

## Agent Definition

| Agent | Instance | Scope | Data Source |
|-------|----------|-------|-------------|
| TechnicalAnalysisAgent | `{userId}:{symbol}` | per-user per-symbol | Alpaca Market Data API |

### Why per-user per-symbol?

- Each user has own Alpaca credentials (BYOK from Phase 2)
- Indicator results cached in agent SQLite per symbol
- Orchestrator spawns one TA agent per watchlist symbol
- `scheduleEvery(60)` during market hours, `scheduleEvery(3600)` after hours

### Pull Model

Agent computes indicators on schedule, stores results in SQLite + writes signals to shared PG. Orchestrator calls:
- `agent.getSignals()` — detected TA signals (RSI oversold, MACD cross, etc.)
- `agent.getIndicators()` — full indicator snapshot (for LLM context)

---

## Goals & Non-Goals

**Goals**:
- Standalone TA agent per user:symbol
- Port TA calculations from mahoraga
- Fetch market data (bars) from Alpaca per-user
- `@callable()` RPC: `getSignals()`, `getIndicators()`, `analyze()`
- Scheduled recalculation via `scheduleEvery()`
- Write TA signals to shared PG `signals` table
- Cache indicators in agent SQLite
- Price chart with indicator overlays
- Signal markers on chart

**Non-Goals**:
- Real-time streaming (WebSocket price updates)
- Custom indicator periods (fixed defaults for MVP)
- Alert triggers (Phase 16)

## Features

### 5.1 TechnicalAnalysisAgent
Per-user per-symbol agent. Fetches bars from Alpaca, computes all indicators, detects signals, writes to shared PG.

### 5.2 Indicator Calculations
SMA (20/50/200), EMA (12/26), RSI (14), MACD (12/26/9), Bollinger (20,2), ATR (14). Ported from mahoraga.

### 5.3 Signal Detection
Auto-detect: RSI oversold/overbought, MACD crossover, BB touch, SMA golden/death cross, volume spike.

### 5.4 Market Data Provider
Get historical bars from Alpaca using user's credentials. Cached in agent SQLite.

### 5.5 Price Chart Component
Candlestick chart using lightweight-charts with indicator overlays.

### 5.6 Indicator Panels
RSI, MACD, ATR, relative volume display cards.

## Type Definitions

```ts
// packages/data-ops/src/agents/ta/types.ts

export interface Bar {
  t: string
  o: number
  h: number
  l: number
  c: number
  v: number
  n: number
  vw: number
}

export interface TechnicalIndicators {
  symbol: string
  timestamp: string
  price: number
  sma_20: number | null
  sma_50: number | null
  sma_200: number | null
  ema_12: number | null
  ema_26: number | null
  rsi_14: number | null
  macd: { macd: number; signal: number; histogram: number } | null
  bollinger: { upper: number; middle: number; lower: number; width: number } | null
  atr_14: number | null
  volume_sma_20: number | null
  relative_volume: number | null
}

export type SignalDirection = "bullish" | "bearish" | "neutral"

export interface TechnicalSignal {
  type: string
  direction: SignalDirection
  strength: number // 0-1
  description: string
}

export interface AnalysisResult {
  symbol: string
  timeframe: string
  indicators: TechnicalIndicators
  signals: TechnicalSignal[]
  bars: Bar[]
}

export type Timeframe = "1Min" | "5Min" | "15Min" | "1Hour" | "1Day"

// Agent state — synced to UI via WebSocket
export interface TAAgentState {
  lastComputeAt: string | null
  symbol: string
  latestPrice: number | null
  signalCount: number
  errorCount: number
  lastError: string | null
}

// RPC interface
export interface TAAgentRPC {
  getSignals(since?: string): Promise<TechnicalSignal[]>
  getIndicators(): Promise<TechnicalIndicators | null>
  analyze(timeframe?: Timeframe): Promise<AnalysisResult>
}
```

## Prerequisites

- **`signals` table**: Phase 4 defines this table but it's in roadmap. Phase 5 extracts the `signals` DDL + Drizzle schema into its own scope (see 005-2-data).
- **`api_credentials` table**: Minimal credentials storage for BYOK Alpaca keys. Full BYOK management UI stays in Phase 2. Phase 5 adds the table + encrypt/decrypt helpers only.
- **Auth middleware**: Uses Better Auth session validation to extract `userId`. No new API key auth — reuses existing Better Auth infra.
