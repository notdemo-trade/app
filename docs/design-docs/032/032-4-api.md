# Phase 32: Proposal Dedup Guards & Minor Data Fixes -- Part 4: API

## No API Changes

All three fixes are internal to the session agent and pipeline orchestrator agent. No Hono endpoints are added, modified, or removed.

---

## Existing Endpoints: Behavioral Changes

### `triggerAnalysis()` Callable (WebSocket RPC)

**Before**: Always runs analysis for every watchlist symbol. Returns `{ threadIds: string[] }` with one thread per symbol.

**After**: Skips symbols with active (pending/approved) proposals. Returns `{ threadIds: string[] }` with threads only for symbols that were actually analyzed. Skipped symbols do not produce threads.

**Consumer impact**: The frontend `triggerAnalysis` call may return fewer `threadIds` than there are watchlist symbols. This is already handled -- the frontend uses the returned `threadIds` to navigate to thread views, and an empty or smaller array simply means fewer threads to display.

### `getThreads()` Callable (WebSocket RPC)

**Before**: May show multiple threads for the same symbol, each with its own pending proposal.

**After**: Same behavior for existing threads. Going forward, new threads for a symbol are only created when no active proposal exists, reducing the frequency of duplicate-symbol threads.

### `getProposals()` Callable (WebSocket RPC)

**No change**. The proposal list still shows all proposals regardless of dedup status. Historical duplicates (created before this fix) remain visible.

## Frontend Server Functions

No changes to any server functions in `apps/user-application/src/core/functions/`.

## WebSocket Messages

The `broadcastThread()` method sends thread updates over the WebSocket connection. Because the dedup guard returns before creating a discussion thread, no WebSocket message is sent for skipped symbols. This is correct -- there is no thread to broadcast.

---

## Observability

The dedup skip is visible through the return value of `triggerAnalysis()`:
- The `threadIds` array will not include an entry for skipped symbols
- The `summary` field for skipped symbols is `Skipped ${symbol}: active proposal already exists`

This summary is not currently exposed to the frontend (only `threadIds` are returned), but it is available for logging within the agent. To add observability, a `console.log` can be added at the skip point:

```ts
if ((existingActive[0]?.cnt ?? 0) > 0) {
    console.log(`[SessionAgent] Skipped analysis for ${symbol}: active proposal exists`);
    return {
        threadId: '',
        summary: `Skipped ${symbol}: active proposal already exists`,
    };
}
```

This log is visible in the Cloudflare Workers logs dashboard and `wrangler tail`.
