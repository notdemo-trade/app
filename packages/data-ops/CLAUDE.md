# data-ops package

Shared DB layer consumed by user-application and data-service apps.

## Stack

- Drizzle ORM + Neon Postgres (serverless)
- Zod v4 for validation
- Better Auth for authentication

## Zod Schema Naming

| Purpose | Suffix | Example |
|---------|--------|---------|
| Domain model | `Schema` | `UserSchema` |
| Request | `RequestSchema` | `UserCreateRequestSchema` |
| Response | `ResponseSchema` | `UserListResponseSchema` |
| Type export | no suffix | `User`, `UserCreateInput` |

## Workflows

**New table:**
1. Add to `src/drizzle/schema.ts`
2. Add relations to `src/drizzle/relations.ts` if needed
3. `pnpm run drizzle:dev:generate`
4. `pnpm run drizzle:dev:migrate`
5. Create queries in `src/queries/{feature}.ts`
6. Create zod schemas in `src/zod-schema/{feature}.ts`
7. `pnpm run build`

**Seed data:** `pnpm run seed:dev`

## Patterns

- Queries use `getDb()` from `database/setup.ts`
- All queries are async, return typed results
- Use `eq`, `count` from drizzle-orm for queries
- Insert with `.returning()` to get created record
- Use `.onConflictDoNothing()` for idempotent inserts

## Don't

- Edit `auth-schema.ts` directly (auto-generated by Better Auth)
- Use `any` type - create interfaces
- Forget to rebuild after changes (`pnpm run build`)
