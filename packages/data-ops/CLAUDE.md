# data-ops package

Shared DB layer consumed by user-application and data-service apps.

## Stack

- Drizzle ORM + Neon Postgres (serverless)
- Zod v4 for validation
- Better Auth for authentication

## Workflows

**New domain:**
1. Create `src/{domain}/` with `table.ts`, `schema.ts`, `queries.ts`, `index.ts`
2. Import table into `src/drizzle/schema.ts`
3. Add relations to `src/drizzle/relations.ts` if needed
4. Add export to `package.json`: `"./{domain}": { "types": "./dist/{domain}/index.d.ts", "default": "./dist/{domain}/index.js" }`
5. `pnpm run drizzle:dev:generate && pnpm run drizzle:dev:migrate`
6. `pnpm run build`

**Seed data:** `pnpm run seed:dev`

## Patterns

- Queries use `getDb()` from `database/setup.ts`
- All queries are async, return typed results
- Use `eq`, `count` from drizzle-orm for queries
- Insert with `.returning()` to get created record
- Use `.onConflictDoNothing()` for idempotent inserts

## Don't

- Edit `auth-schema.ts` directly (auto-generated by Better Auth)
- Use `any` type ‚Äî create interfaces
- Forget to rebuild after changes (`pnpm run build`)
- Use old `@repo/data-ops/queries/*` or `@repo/data-ops/zod-schema/*` paths
