# i18n Rules

## Library

`use-intl`. Use `useTranslations()` and `useLocale()` hooks in components.

## Adding Translatable Strings

1. Add key to both `i18n/messages/en.json` and `i18n/messages/pl.json`
2. Use flat dot-separated keys: `"section.subsection.label"`
3. Use ICU message format for plurals/variables: `"{count, plural, one {# item} other {# items}}"`
4. In component: `const t = useTranslations()` then `t("section.label")`

## Key Naming Convention

```
{section}.{subsection}.{element}

nav.features          # navigation item
hero.title.part1      # hero section
faq.beginner.q1       # FAQ question
faq.beginner.a1       # FAQ answer
meta.title            # page meta
```

## Never Translate

- Brand name "notdemo.trade"
- External service names (OpenAI, Anthropic, etc.)
- URL slugs and route paths

## Locale Strategy

- **Public pages**: locale from URL path. No prefix = EN (default), `/pl/` prefix = Polish
- **Auth/dashboard pages** (`/app/*`, `/api/*`): locale from cookie, never URL-prefixed
- Ignored paths: `/app`, `/api`, `/rpc`, `/_auth`

## Hydration Safety

- Never use `typeof window` or `typeof document` to determine locale
- Use `createIsomorphicFn()` for server/client locale detection
- Locale flows from `beforeLoad` -> route context -> `IntlProvider`
- Messages loaded via `useSuspenseQuery` -- never render without messages

## Links

Standard `<Link>` works -- router `rewrite` handles locale prefix automatically. No wrapper needed.

## New Component Checklist

When creating a component with user-facing strings:
1. Import `useTranslations` from `use-intl`
2. Add keys to both locale JSON files
3. Never hardcode user-facing strings

## Translation Files

- Location: `apps/user-application/src/i18n/messages/`
- Format: flat JSON, one file per locale (`en.json`, `pl.json`)
- Loaded via RPC (`createServerFn`), cached with `staleTime: Infinity`
- Keep both locale files in sync -- every key in `en.json` must exist in `pl.json`

## Config

- Types/constants in `i18n/core/shared.ts` (`Locale`, `SUPPORTED_LOCALES`, `DEFAULT_LOCALE`)
- Rewrite functions in `i18n/core/client.ts`
- Middleware in `i18n/core/server.ts`
- Design doc: `docs/019-i18n-multilingual-support.md`
