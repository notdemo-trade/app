# Better Auth Rules

## Configuration

- Configure in `config/auth.ts`, export auth instance
- Never edit `auth-schema.ts` (auto-generated by `better-auth:generate`)
- Use plugins for extended features (2FA, OAuth, etc.)

```ts
import { betterAuth } from 'better-auth'

export const auth = betterAuth({
  database: { ... },
  plugins: [ ... ],
  session: {
    expiresIn: 60 * 60 * 24 * 7, // 7 days
    updateAge: 60 * 60 * 24,     // refresh daily
  },
})

export type Auth = typeof auth
```

## Database Integration

- Use Drizzle adapter for this project
- Run `pnpm better-auth:generate` after config changes
- Migrations handled via Drizzle

## Session Management

- Configure `expiresIn` and `updateAge` appropriately
- Use cookie caching for serverless (reduces DB hits)
- For sensitive ops, check session freshness

```ts
session: {
  cookieCache: {
    enabled: true,
    maxAge: 5 * 60, // 5 min cache
  },
}
```

## Extending Schema

- Add custom fields via `additionalFields`
- Use `input: false` for server-only fields

```ts
user: {
  additionalFields: {
    role: {
      type: 'string',
      required: true,
      defaultValue: 'user',
      input: false, // server assigns, not user
    },
  },
}
```

## Security

- Never expose auth secrets in client code
- Use HTTP-only cookies for session tokens
- Implement rate limiting on auth endpoints
- Revoke sessions on password change

## Client Integration

- Export auth client config for frontend
- Use typed client from better-auth/react
